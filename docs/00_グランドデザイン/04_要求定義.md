## 2. 優先度の定義

| 優先度     | 定義                                               | 対応フェーズ     |
| ---------- | -------------------------------------------------- | ---------------- |
| **Must**   | システム稼働に必須。ないと業務が回らない           | フェーズ1（MVP） |
| **Should** | 強く期待されている。なくても業務は回るが不満が残る | フェーズ1〜2     |
| **Could**  | あれば嬉しい。優先度は低い                         | フェーズ2以降    |

## 5. 要求のステータス管理

| ステータス | 定義                             |
| ---------- | -------------------------------- |
| 提案       | ステークホルダーから挙がった段階 |
| 承認       | プロジェクトオーナーが対応を承認 |
| 実装中     | 対応する機能を開発中             |
| 完了       | 機能がリリース済み               |
| 却下       | スコープ外として却下             |
| 延期       | 将来フェーズに延期               |
