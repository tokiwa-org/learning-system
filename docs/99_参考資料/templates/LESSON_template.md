# LESSON（学習コンテンツ）テンプレート

## 概要

LESSSONは座学形式の学習コンテンツです。
概念の理解、知識の習得を目的とします。

---

## メタ情報

```yaml
itemType: LESSON
estimatedMinutes: 8-15  # 標準は10分
contentLength: 1500-2500  # 文字数目安
```

---

## 構成パターン

### パターンA: 「なぜ学ぶのか」（Step 1用）

カリキュラムの最初に配置し、学習動機を高めるコンテンツ。

```markdown
## [トピック]を学ぶ理由

[導入文：なぜ今このスキルが重要か（2-3文）]

---

## 現在の状況

| 指標 | 数値 |
|------|------|
| 指標1 | 数値（出典明記） |
| 指標2 | 数値 |
| 指標3 | 数値 |

### 業界動向

[OWASP、Gartner、IDC等の権威ある情報源からの引用]

> 「引用文」— 出典

[事例の概要と影響]
```

## ノイズレベル別の調整

### MINIMAL（L1向け）

- 完全に整理された情報
- 正解のパターンを明確に提示
- 「なぜ」の説明を丁寧に
- コード例はシンプルに

### LOW（L1→L2）

- 一部の情報を「調べてみましょう」に
- 複数の方法を紹介（推奨は明示）
- 軽微な注意点を追加

### MEDIUM（L2→L3）

- トレードオフの説明を追加
- 「ケースバイケース」の判断基準
- レガシーコードとの比較

### HIGH（L3→L4）

- 複数の選択肢を並列提示
- 「正解がない」問題の提示
- 実務での判断基準

### MAXIMUM（L4→L5）

- 矛盾する情報源の提示
- コンテキストによる判断
- 自ら調査・判断する設問

## AI生成プロンプト

```
以下の仕様でLESSONコンテンツを生成してください：

【基本情報】
- タイトル: {title}
- stepTitle: {stepTitle}（命令形）
- stepContext: {context}
- パターン: {A|B|C}（なぜ学ぶ|基礎知識|対策方法）
- 想定時間: {minutes}分
- ノイズレベル: {noiseLevel}

【内容要件】
- 最新の統計データを含める（2024-2025年）
- コード例は TypeScript で記述
- 表やリストを活用して視認性を高める
- 各セクションは「---」で区切る

【文字数】
- 総文字数: 約{char_count}文字
- 1セクションあたり: 200-400文字

【出力形式】
Markdown形式で出力
```
