# カリキュラムコンテンツ構成テンプレート

## 概要

AI生成時に使用するカリキュラムコンテンツの標準構成。
一貫した品質と学習効果を確保するためのテンプレート。

---

## 1. カリキュラム全体構成

### 1.1 メタ情報

```yaml
curriculum:
  name: "カリキュラム名"
  missionTitle: "〇〇を解決せよ"  # 命令形のミッションタイトル
  missionSummary: "1-2文の概要"
  backgroundStory: "学習者が置かれる状況設定（100-200文字）"
  missionObjective: "このカリキュラムで達成できること"

  metadata:
    targetGrade: ["G2", "G3"]  # 対象職級
    learningPhase: "INTERMEDIATE"  # BEGINNER/INTERMEDIATE/ADVANCED
    noiseLevel: "LOW"  # MINIMAL/LOW/MEDIUM/HIGH/MAXIMUM
    estimatedHours: 2
    category: "共通スキル"  # or "専門スキル"
    tags: ["セキュリティ", "Web開発", "データベース"]
```

### 1.2 標準的なアイテム構成

| # | itemType | 内容 | 目安時間 | テンプレート |
|---|----------|------|---------|-------------|
| 1 | LESSON | なぜ学ぶのか（現状・統計・事例） | 8-15分 | パターンA |
| 2 | LESSON | 基礎知識（概念・仕組み） | 8-15分 | パターンB |
| 3 | LESSON | 対策・解決方法 | 8-15分 | パターンC |
| 4 | EXERCISE | ハンズオン演習（基礎） | 15-30分 | パターンA |
| 5 | QUIZ | 理解度確認 | 5-10分 | - |
| 6 | EXERCISE | 実践演習（応用） | 15-30分 | パターンB |

**総所要時間**: 約1.5〜2時間（スタンダードカリキュラム）

## 3. ノイズレベル別の内容調整

### 3.1 MINIMAL（L1向け）

- 完全に整理された情報
- 正解の型を明確に提示
- エラーのない環境
- ステップバイステップの詳細な指示

### 3.2 LOW（L1→L2）

- 情報の一部が不完全（10%程度）
- 「調べてみましょう」のプロンプト追加
- 軽微なエラーメッセージへの対処

### 3.3 MEDIUM（L2→L3）

- 技術的負債のあるコード
- 複数の解決アプローチ提示
- ドキュメントと実装の軽微な乖離

### 3.4 HIGH（L3→L4）

- 選択肢の過剰（どれも正しそう）
- トレードオフの判断が必要
- 不完全な要件定義

### 3.5 MAXIMUM（L4→L5）

- コンテキストの揺らぎ
- 矛盾する情報源
- 正解のない問題

## 4. AI生成時のプロンプト

### 4.1 カリキュラム全体生成

```
以下の仕様でカリキュラム全体を生成してください：

【カリキュラム情報】
- タイトル: {title}
- ミッションタイトル: {missionTitle}（命令形）
- 対象職級: {targetGrades}
- ノイズレベル: {noiseLevel}
- 関連スキル: {relatedSkills}
- 総時間: {totalHours}時間

【アイテム構成】
標準構成（6ステップ）に従って生成：
1. LESSON: なぜ学ぶのか（パターンA）
2. LESSON: 基礎知識（パターンB）
3. LESSON: 対策方法（パターンC）
4. EXERCISE: 基礎演習（パターンA）
5. QUIZ: 理解度確認
6. EXERCISE: 実践演習（パターンB）

【要件】
- 各テンプレートファイルの形式に従う
- 最新の統計データ・事例を含める（2024-2025年）
- コード例は TypeScript で記述
- ノイズレベルに応じた難易度調整
```

### 4.2 個別アイテム生成

各アイテムタイプの生成プロンプトは、個別テンプレートファイルを参照してください：

- LESSON: [templates/LESSON_template.md](./templates/LESSON_template.md)
- EXERCISE: [templates/EXERCISE_template.md](./templates/EXERCISE_template.md)
- QUIZ: [templates/QUIZ_template.md](./templates/QUIZ_template.md)

## 5. 品質チェックリスト

### コンテンツ品質

- [ ] 最新の統計データ・事例が含まれている
- [ ] コード例が正しく動作する
- [ ] 専門用語に説明がある
- [ ] 適切な長さ（LESSONは8-15分で読める量）

### 学習効果

- [ ] 学習目標が明確
- [ ] 段階的に難易度が上がる
- [ ] 実践的な演習がある
- [ ] 自己評価できる仕組みがある

### ノイズレベル適合

- [ ] 対象職級に適切な難易度
- [ ] 指定されたノイズタイプが適用されている
- [ ] 過度に難しすぎない / 簡単すぎない

### 時間設計

- [ ] 各アイテムのestimatedMinutesが設定されている
- [ ] カリキュラム全体で目標時間内に収まる
- [ ] LESSON:EXERCISE:QUIZの比率が適切（50-60%:30-40%:10%）

## 6. 例：SQLインジェクションカリキュラム

### メタ情報

```yaml
curriculum:
  name: "SQLインジェクション対策"
  missionTitle: "SQLインジェクション攻撃を阻止せよ"
  missionSummary: "本番システムへの不審なアクセスを調査し、セキュリティ脆弱性を修正する"
  backgroundStory: "あなたは中堅エンジニアとして、本番システムのセキュリティ強化プロジェクトにアサインされました。突然、セキュリティチームから緊急連絡が入りました。「不審なアクセスログを検出。SQLインジェクション攻撃の可能性がある。至急対応してほしい」"
  missionObjective: "SQLインジェクションの仕組みを理解し、脆弱なコードを特定・修正できるようになる"

  metadata:
    targetGrade: ["G2", "G3"]
    learningPhase: "INTERMEDIATE"
    noiseLevel: "LOW"
    estimatedHours: 1.5
    category: "共通スキル"
    tags: ["セキュリティ", "Web開発", "データベース"]
```

### アイテム構成

| # | itemType | title | stepTitle | 時間 |
|---|----------|-------|-----------|------|
| 1 | LESSON | なぜSQLインジェクションを学ぶのか | 攻撃の仕組みを理解せよ | 10分 |
| 2 | LESSON | SQLインジェクションの仕組み | 攻撃パターンを把握せよ | 15分 |
| 3 | LESSON | SQLインジェクション対策方法 | 対策方法を学べ | 15分 |
| 4 | EXERCISE | 脆弱なコードを見つける | 脆弱なコードを特定せよ | 20分 |
| 5 | QUIZ | 理解度確認クイズ | 知識を確認せよ | 8分 |
| 6 | EXERCISE | コードリファクタリング | 実際にコードを修正せよ | 25分 |

**合計**: 約1.5時間
