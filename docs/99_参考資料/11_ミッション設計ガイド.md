## 教育的ノイズについて

> 参照: [09_教育的ノイズ.md](./09_教育的ノイズ.md)

### 基本概念

**教育的ノイズ**とは、実務で直面する「不確実性」や「複雑性」を、成長に寄与する形でカリキュラムに組み込むことです。

```
SN比 = 信号（教えたい本質） ÷ 雑音（意図的な不確実性）
```

- **信号が雑音に完全に埋もれる** → 学習は停止する
- **雑音がゼロ** → 学習者の「フィルタリング性能」が育たない

### ノイズレベルの定義

| ノイズレベル | 教育的意味 | 対象フェーズ | 設計方針 |
|-------------|-----------|-------------|---------|
| **MINIMAL** | ノイズ最小限 | Phase 1（ジュニア・L1） | 「正解の型」を脳に刻む。整理された教材で基礎を固める |
| **MEDIUM** | 中程度のノイズ | Phase 2（ミドル・L2-L3） | 実務に近い複雑さを導入。デバッグ・リファクタリングを通じて適応力を養う |
| **HIGH** | 高いノイズ | Phase 3（シニア・L4-L5） | 曖昧な要件、複数の選択肢、正解のない状況を提示 |

### ノイズ設計マトリクス

| カテゴリ | 具体的な形態 | 期待される教育効果 | 推奨レベル |
|---------|-------------|-------------------|-----------|
| **情報の不完全性** | 意図的な考慮漏れがある仕様書、古いドキュメント | 「批判的思考」の育成 | L1〜L3 |
| **技術的負債** | 密結合なレガシーコード、一貫性のない命名規則 | 「解読力と改善力」の育成 | L2〜L3 |
| **選択肢の過剰** | 複数の解決策の提示、正解のない技術選定 | 「意思決定・トレードオフ判断力」の育成 | L3〜L4 |
| **コンテキストの揺らぎ** | 途中で変わる優先順位、曖昧な要望 | 「柔軟性と適応力」の育成 | L4〜L5 |
| **検索環境の汚染** | 似た名前の別技術、廃止されたAPIの検索結果 | 「情報リテラシー」の育成 | 全レベル |

### noiseLevelの配置場所（重要）

> 参照: [06_ADR_ノイズレベル配置設計](../02_基本設計/06_ADR_ノイズレベル配置設計.md)

**教育的ノイズレベル (`noiseLevel`) は `Curriculum`（カリキュラム）エンティティに配置します。**

```typescript
interface Curriculum {
  noiseLevel: NoiseLevel;      // MINIMAL | LOW | MEDIUM | HIGH | MAXIMUM
  noiseTypes?: NoiseType[];    // 具体的なノイズの種類
  noiseDesignNotes?: string;   // 査閲者向けノイズ設計説明
}
```

#### 設計上の理由

同じスキル（例：データベース）でも、学習者のレベルによって異なるノイズ濃度で教えるべきです：

| 学習者 | カリキュラムのnoiseLevel | 理由 |
|-------|------------------------|------|
| L1新人 | MINIMAL | まず「正解の型」を習得させる |
| L3中堅 | MEDIUM | 技術的負債やデバッグを含める |
| L5シニア | HIGH | 曖昧な要件、選択肢の過剰を提示 |

#### Neo4jオントロジーのnoiseLevelについて

Neo4jの`RoadmapItem.noiseLevel`は**参照情報として残す**が、カリキュラム生成時は`Curriculum.noiseLevel`を使用します。

RoadmapItem.noiseLevelは「スキルの複雑性の目安」として有用：

| noiseLevel | スキル | カテゴリ | 意味 |
|------------|--------|---------|------|
| **MINIMAL** | OS, ネットワーク, データベース, HTML/CSS, Git | IT基本 | 基礎的で複雑性が低いスキル |
| **MEDIUM** | AWS, Docker, Kubernetes, Terraform | クラウド・コンテナ・IaC | 中程度の複雑性を持つスキル |
| **HIGH** | メイン開発言語, マルチクラウド, アーキテクチャ設計 | プログラミング・設計 | 本質的に複雑なスキル |

### 良質なノイズ vs 無益なノイズ

#### ✅ 成長につながるノイズ（実務で頻出するパターン）

- 仕様変更への対応
- 古いライブラリの刷新
- トレードオフの判断
- 曖昧な要件からの仕様抽出
- 複数技術からの選定

#### ❌ ただのストレスになるノイズ（避けるべき）

- システムの不具合で課題が進まない
- 説明不足すぎて手が動かせない
- 評価基準が不透明
- 意図のない複雑さ
- 学習と無関係な障害

## レベル別スキル定義

### レベル概要

| レベル | 対象者 | ノイズ濃度 | 特徴 |
|-------|-------|-----------|------|
| L0 | 未経験者 | - | 基本操作ができる |
| L1 | 新人 | **MINIMAL** | 指示に従って実行できる。「正解の型」を学ぶ |
| L2 | 一人前 | **MINIMAL→MEDIUM** | 自律的に業務遂行できる。ノイズ耐性を徐々に上げる |
| L3 | 中堅 | **MEDIUM** | チームをリードできる。実務的な複雑さに対処する |
| L4 | シニア | **MEDIUM→HIGH** | 組織に影響を与えられる。曖昧な状況で判断する |
| L5 | エキスパート | **HIGH** | 組織文化を変革できる。正解のない問題に取り組む |

## L1ミッション設計

### L1の特徴

- **対象**: 新卒・第二新卒、未経験からの転職者
- **ノイズ濃度**: **MINIMAL**（正解の型を学ぶ）
- **目標**: 指示を受けて正確に実行できる
- **支援**: 先輩・メンターのサポートあり

### L1ノイズ設計方針

```
⚠️ 注意: L1でノイズが多いと、基礎が固まらずに「動けばいい」という悪い癖がつく可能性あり
```

- まずは「正解の型」を脳に刻む
- 教材は整理された状態で提供
- ただし「1箇所だけあえて古い書き方」を混ぜる等、最小限のノイズは有効
- 有識者とのレビューで「なぜこれではダメなのか」を議論させる

### L1スキル一覧（カテゴリ別）

#### IT基本（noiseLevel: MINIMAL）

| スキル | L1定義 | ミッションテーマ案 |
|--------|--------|-------------------|
| OS | シェルスクリプトを書ける | 日常作業を自動化せよ |
| ネットワーク | ネットワーク設定・トラブルシューティングができる | 接続障害を復旧させよ |
| データベース | JOINやサブクエリを使った複雑なSQLを書ける | 売上データの謎を解明せよ |
| HTML/CSS | レスポンシブデザイン・Flexbox/Gridを使える | モバイル対応サイトを構築せよ |
| Git/バージョン管理 | ブランチ戦略・コンフリクト解決ができる | ブランチの衝突を収束させよ |
| 生成AI | ChatGPT/Claudeで基本的な質問・タスクができる | AIを開発の味方につけよ |

#### プログラミング

| スキル | L1定義 | ミッションテーマ案 |
|--------|--------|-------------------|
| メイン開発言語 | 基本文法・標準ライブラリを理解している | 言語の基礎を完全習得せよ |
| サブ開発言語 | 基本文法を理解し、簡単なコードが書ける | 第二言語の扉を開けよ |

#### コンテナ・クラウド（noiseLevel: MEDIUM → L1では基礎のみ）

| スキル | L1定義 | ミッションテーマ案 |
|--------|--------|-------------------|
| Docker | Dockerfileを理解し、基本的なコンテナを作成できる | アプリをコンテナに閉じ込めよ |
| Kubernetes | 基本概念(Pod、Service等)を理解している | K8sの世界に踏み込め |
| AWS | 基本サービス(EC2、S3、RDS)を理解している | クラウドの基礎を制覇せよ |
| Terraform | 基本構文を理解し、既存コードを読める | インフラをコードで読み解け |

#### セキュリティ

| スキル | L1定義 | ミッションテーマ案 |
|--------|--------|-------------------|
| セキュアコーディング | 基本的な脆弱性(SQLi、XSS等)を理解している | SQLインジェクション攻撃を阻止せよ |
| 認証・認可 | OAuth/JWTの基本を理解している | 認証の仕組みをマスターせよ |
| DevSecOps | DevSecOpsの概念を理解している | セキュリティをDevOpsに組み込め |

#### 品質・テスト

| スキル | L1定義 | ミッションテーマ案 |
|--------|--------|-------------------|
| テスト・検証 | 基本的なテストを実施できる | バグを捕捉するテストを書け |
| コードレビュー | レビューコメントを理解し、修正できる | レビュー指摘を完璧に修正せよ |
| 品質基準遵守 | 既存の品質基準を遵守できる | 品質基準をクリアせよ |

#### 生成AI

| スキル | L1定義 | ミッションテーマ案 |
|--------|--------|-------------------|
| ChatGPT/Claude | 基本的な質問・タスクに活用できる | AIと対話する技術を習得せよ |
| GitHub Copilot | コード補完を利用できる | AIペアプログラマーを使いこなせ |
| プロンプトエンジニアリング | 基本的なプロンプトを作成できる | 効果的なプロンプトを設計せよ |

#### ソフトスキル

| スキル | L1定義 | ミッションテーマ案 |
|--------|--------|-------------------|
| 進捗報告 | 自分の進捗を報告できる | 進捗を正確に可視化せよ |
| エスカレーション | 重大な問題を報告できる | 適切なタイミングで助けを求めよ |
| 見積り | サポートを受けて見積もりできる | 作業時間を見積もれ |
| 期限遵守 | 明確な期限があれば守れる | 期限内に完遂せよ |

### L1ミッション詳細例

#### ミッション1: シェルスクリプトで作業を自動化せよ

```yaml
mission:
  title: "日常作業を自動化せよ"
  category: "IT基本"
  target_skill: "OS - シェルスクリプト"
  noise_level: "MINIMAL"  # 正解の型を学ぶ

background_story: |
  あなたは新卒エンジニアとして開発チームに配属されました。
  先輩から「毎朝のログファイル整理、手動でやってる？
  それ、スクリプト化すれば30秒で終わるよ」と言われました。
  日々の単純作業を自動化して、本質的な開発に集中しましょう。

objective: |
  シェルスクリプトの基本を習得し、日常業務を自動化できるようになる

noise_design: |
  - 基本的にはクリーンな教材を提供
  - STEP 4で「あえて非効率な実装例」を1つ混ぜ、改善点を議論させる

steps:
  - number: 1
    title: "シェルの基本コマンドを習得せよ"
    context: "ls, cd, grep, find など基本コマンドをマスターします"
    estimated_minutes: 15
    noise: "なし（正解の型を学ぶ）"

  - number: 2
    title: "変数と制御構文をマスターせよ"
    context: "if, for, while を使った条件分岐とループを学びます"
    estimated_minutes: 20
    noise: "なし"

  - number: 3
    title: "ファイル操作スクリプトを作成せよ"
    context: "ファイルの読み書き、移動、削除を自動化します"
    estimated_minutes: 25
    noise: "なし"

  - number: 4
    title: "【演習】ログローテーションスクリプトを完成させよ"
    context: "古いログを圧縮・アーカイブするスクリプトを作成します"
    estimated_minutes: 30
    noise: "MINIMAL - サンプルに1箇所だけ非推奨な書き方を混入"

  - number: 5
    title: "【実践】自分の日常業務を自動化せよ"
    context: "実際の業務で繰り返している作業を特定し、自動化します"
    estimated_minutes: 40
    noise: "MINIMAL - 実際の業務は多少の曖昧さを含む"
```

#### ミッション2: SQLインジェクション攻撃を阻止せよ

```yaml
mission:
  title: "SQLインジェクション攻撃を阻止せよ"
  category: "セキュリティ"
  target_skill: "セキュアコーディング"
  noise_level: "MINIMAL"

  本番システムで不審なアクセスログを検出しました。

  - STEP 3の演習で「明らかに脆弱なコード」と「一見安全そうだが脆弱なコード」を混ぜる

steps:
  - number: 1
    title: "攻撃の仕組みを理解せよ"
    context: "SQLインジェクション攻撃がどのように行われるか学びます"
    estimated_minutes: 10
    noise: "なし"

  - number: 2
    title: "対策方法を学べ"
    context: "プリペアドステートメントなど対策方法を学びます"
    estimated_minutes: 15
    noise: "なし"

  - number: 3
    title: "【演習】脆弱なコードを特定せよ"
    context: "サンプルコードから脆弱性を見つけ出します"
    estimated_minutes: 20
    noise: "MINIMAL - 難易度の異なる脆弱性を複数混入"

  - number: 4
    title: "知識を確認せよ"
    context: "クイズ形式で理解度をチェックします"
    estimated_minutes: 10
    noise: "なし"

  - number: 5
    title: "【実践】コードを修正せよ"
    context: "脆弱なコードを安全なコードにリファクタリングします"
    estimated_minutes: 25
    noise: "MINIMAL"
```

#### ミッション3: ブランチの衝突を収束させよ

  - number: 4

## L2ミッション設計

### L2の特徴

- **対象**: 1〜3年目の一人前エンジニア
- **ノイズ濃度**: **MINIMAL→MEDIUM**（徐々にノイズ耐性を上げる）
- **目標**: 自分で考えて設計・構築ができる
- **支援**: 必要に応じてレビュー・相談

### L2ノイズ設計方針

```
「動くが保守性が低いノイズだらけのコード」を渡し、
「質の高い状態」へ昇華させるプロセスを課題にする
```

- L1で学んだ「正解の型」を基準に、ノイズを識別させる
- デバッグ・リファクタリングを通じて適応力を養う
- 自分でノイズをフィルタリングし、最適解を導き出す力を育成

### L2スキル一覧（カテゴリ別）

#### IT基本・プログラミング

| スキル | L2定義 | ミッションテーマ案 | ノイズ設計 |
|--------|--------|-------------------|-----------|
| OS | システム管理・トラブルシューティングができる | システム障害を復旧させよ | 不完全なログ情報から原因を特定 |
| ネットワーク | ネットワーク設計・セキュリティ設定ができる | セキュアなネットワークを構築せよ | 複数の設計パターンから選択 |
| データベース | データベース設計・インデックス設計ができる | 最適なDB設計を完成させよ | 非正規化vs正規化のトレードオフ |
| メイン開発言語 | 業務でコードを書き、レビューを受けて修正できる | プロダクションコードを書け | レガシーコードとの整合性 |

#### コンテナ・クラウド・IaC（noiseLevel: MEDIUM）

| スキル | L2定義 | ミッションテーマ案 | ノイズ設計 |
|--------|--------|-------------------|-----------|
| Docker | 最適化されたイメージをビルド・運用できる | 本番レディなイメージを作成せよ | 非効率なDockerfileを改善させる |
| Kubernetes | マニフェストを書き、アプリをデプロイできる | K8sにアプリをデプロイせよ | 複数のデプロイ戦略から選択 |
| AWS | サービスを構築・運用できる | AWSでシステムを構築せよ | コストvs可用性のトレードオフ |
| Terraform | リソースを定義し、デプロイできる | インフラをコード化せよ | 既存の手動構築環境をコード化 |

#### CI/CD・DevOps

| スキル | L2定義 | ミッションテーマ案 | ノイズ設計 |
|--------|--------|-------------------|-----------|
| GitHub Actions | ビルド・テスト・デプロイパイプラインを構築できる | CI/CDパイプラインを構築せよ | 複数のワークフロー設計から選択 |
| SRE原則 | エラーバジェット・トイル削減を実践できる | トイルを削減せよ | 優先順位付けが必要な複数のトイル |

#### セキュリティ

| スキル | L2定義 | ミッションテーマ案 | ノイズ設計 |
|--------|--------|-------------------|-----------|
| セキュアコーディング | セキュアなコードを実装できる | セキュアなAPIを実装せよ | 既存の脆弱なコードを改修 |
| 認証・認可 | 認証・認可機能を実装できる | OAuth認証を実装せよ | 複数の認証方式から選択 |

#### 生成AI
