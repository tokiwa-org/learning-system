# L3向け 10ヶ月カリキュラム（年間200時間）

## 概要

| 項目 | 内容 |
|-----|------|
| 対象レベル | L3（中堅）→ L4（シニア）への成長 |
| 総学習時間 | 200時間（20時間 × 10ヶ月） |
| 形式 | ミッション型カリキュラム |
| ノイズレベル | MEDIUM（中程度） |

## 月別詳細

### 月1: システム全体のアーキテクチャを設計しよう（20時間）

```yaml
mission:
  title: "システム全体のアーキテクチャを設計しよう"
  roadmap:
    skills:
      - name: "アーキテクチャ設計"
        category: "技術選定・設計力"
        achievement: "システム全体のアーキテクチャを設計できる"
      - name: "技術スタックの選定"
        category: "技術選定・設計力"
        achievement: "プロジェクト要件に最適な技術を選定できる"
```

| # | ステップタイトル | 種別 | 時間 |
|---|----------------|------|------|
| 1 | ビジネス要件からアーキテクチャを導こう | LESSON | 3h |
| 2 | 非機能要件を明確化しよう | LESSON + EXERCISE | 4h |
| 3 | アーキテクチャ選択肢を比較検討しよう | EXERCISE | 4h |
| 4 | トレードオフを分析して決定を下そう | EXERCISE | 4h |
| 5 | アーキテクチャ設計書を完成させよう | EXERCISE | 3h |
| 6 | 設計レビューを通過しよう | QUIZ + EXERCISE | 2h |

**背景ストーリー**:
新規事業のシステム構築を任された。経営層からは「5年先を見据えた柔軟な設計を」と要望されている。様々な選択肢がある中で、最適なアーキテクチャを設計しなければならない。チームの未来を左右する重要な決断だ。
