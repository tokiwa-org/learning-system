# L2向け 10ヶ月カリキュラム（年間200時間）

## 概要

| 項目 | 内容 |
|-----|------|
| 対象レベル | L2（一人前）→ L3（中堅）への成長 |
| 総学習時間 | 200時間（20時間 × 10ヶ月） |
| 形式 | ミッション型カリキュラム |
| ノイズレベル | MINIMAL→MEDIUM |

## 月別詳細

### 月1: 設計パターンを武器にしよう（20時間）

```yaml
mission:
  title: "設計パターンを武器にしよう"
  roadmap:
    skills:
      - name: "DDD戦術的設計"
        category: "設計思想"
        achievement: "集約・リポジトリパターンを実装できる"
      - name: "クリーンアーキテクチャ"
        category: "設計思想"
        achievement: "レイヤー分離の概念を理解している"
```

| # | ステップタイトル | 種別 | 時間 |
|---|----------------|------|------|
| 1 | DDDの戦術パターンを理解しよう | LESSON | 3h |
| 2 | エンティティと値オブジェクトを実装しよう | LESSON + EXERCISE | 4h |
| 3 | 集約とリポジトリを構築しよう | EXERCISE | 4h |
| 4 | クリーンアーキテクチャの層を分けよう | LESSON + EXERCISE | 4h |
| 5 | 依存性逆転を適用しよう | EXERCISE | 3h |
| 6 | 設計パターン適用演習 | QUIZ + EXERCISE | 2h |

**背景ストーリー**:
プロジェクトのコードが肥大化し、変更が困難になってきた。「設計を見直したい」という声が上がる中、設計パターンを武器にしてコードベースを改善せよ。
