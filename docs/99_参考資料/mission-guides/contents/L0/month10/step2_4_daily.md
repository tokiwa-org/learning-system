## 日報テンプレート（月6の復習）

```markdown
# 日報 - YYYY年MM月DD日（曜日）

## 基本情報
- 作成者: ○○
- プロジェクト: 社内ツール紹介ページ作成

## 今日やったこと
- [ ] 作業1
- [ ] 作業2

- [ ] 作業2

## 困っていること / 相談事項
- （あれば記載）

## 所感
（一言コメント）
```

## Part 1: 日報の作成

### タスク 1-1: Day 1 の日報ファイルを作成

以下の内容を参考に、自分の言葉で日報を作成してください。

```bash
cat > docs/daily/day1.md << 'EOF'
# 日報 - Day 1（月曜日）

## 基本情報
- 作成者: （あなたの名前）
- プロジェクト: 社内ツール紹介ページ作成

## 今日やったこと
- [x] 依頼内容の確認、要件整理
- [x] 不明点を田中さんに質問、回答を得た
- [x] 作業計画の作成（5日間のスケジュール）
- [x] プロジェクトフォルダの作成
- [x] Gitリポジトリの初期化、初回コミット
- [x] .gitignoreの作成
- [x] SQLiteデータベースの作成
- [x] テーブル定義（tools テーブル）
- [x] サンプルデータ10件の投入

## 明日やること
- [ ] SQLでデータの取得・分析
- [ ] カテゴリ別の集計
- [ ] ツールURLの疎通確認

## 困っていること / 相談事項
- 特になし。計画通り進められた。

## 所感
初めて一人で仕事を任されて緊張したが、まずは環境構築を完了できた。
これまで学んだことが活きていると実感した。
明日はデータの取得・分析に取り組む。
EOF
```

### タスク 1-2: 日報の確認

```bash
cat docs/daily/day1.md
```

内容を読み返して、以下をチェックしてください。

| チェック項目 | OK? |
|-------------|-----|
| やったことが具体的に書けているか | [ ] |
| 明日やることが明確か | [ ] |
| 困っていることを正直に書けているか | [ ] |
| 誤字脱字がないか | [ ] |

## Part 2: 日報をコミット

### タスク 2-1: コミット

```bash
cd ~/projects/internal-tools-page

git add docs/daily/day1.md
git commit -m "Day 1 日報を追加"
```

### タスク 2-2: 履歴の確認

```bash
git log --oneline
```

期待される出力（例）：
```
jkl3456 Day 1 日報を追加
ghi9012 サンプルデータ投入用SQLスクリプトを追加
def5678 データベーステーブル定義のSQLスクリプトを追加
abc1234 プロジェクトの初期構造を作成
```

4つのコミットが表示されていれば、Day 1の作業はすべて完了です。

---

## 日報を書くときのコツ（月6の復習）

### コツ1: 具体的に書く

```
悪い例: データベースの作業をした
良い例: SQLiteでtoolsテーブルを作成し、サンプルデータ10件を投入した
```

### コツ2: 「困っていること」は正直に

困っていることがないなら「特になし」でOK。でも、もし不安があれば正直に書く。

```
例: CSSのレイアウトに自信がないので、明日の作業で時間がかかるかもしれない。
```

### コツ3: 所感は一言でも書く

日報に人間味を持たせる。先輩も読んでいて安心できる。
