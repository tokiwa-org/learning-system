## これが初仕事だ

### ミッション概要

| 項目 | 内容 |
|------|------|
| 依頼者 | 先輩の田中さん |
| 期限 | 1週間（5営業日） |
| 成果物 | 社内ツール紹介ページ + 完了報告書 |
| 公開先 | GitHub |
| 報告 | 毎日の日報 + 最終週報 |

### 具体的な作業内容

1. **データベースから社内ツールの情報を取得してまとめる**
2. **HTML/CSSで紹介ページを作成する**
3. **Gitで管理してGitHubにプッシュする**
4. **作業の進捗を日報で報告する**
5. **完了報告書を書く**

## 初仕事を成功させるポイント

### 1. まず全体像を把握する

いきなり作業を始めない。まず「何を作るのか」「どのくらいかかるのか」を理解する。

### 2. わからないことは早めに聞く

「聞いてもいいけど、まずは自分で考えて」と言われたが、調べても解決しないことは早めに質問する。

### 3. こまめに記録する

日報で毎日の進捗を報告する。困っていることも正直に書く。

### 4. 完璧を目指さない

初仕事で100点は求められていない。「完遂すること」が最も大切。

Day 1: 要件理解 → 環境構築 → 日報

Day 4: セルフレビュー → 修正 → 日報
Day 5: 報告書作成 → 最終確認 → 週報
