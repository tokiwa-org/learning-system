## 良い質問の3点セット

### 基本フォーマット

```
1. やりたいこと（目的）
   「〇〇をしたいのですが、」

2. 現状（問題）
   「△△という状態になっています。」

3. 試したこと（努力）
   「□□を試しましたが解決しませんでした。」
```

### 具体例

**悪い質問:**

```
「動きません。助けてください。」
```

→ 何が動かないのか、何を試したのかわからない。

**良い質問:**

```
「ログインフォームの送信処理を実装したいのですが、（目的）
送信ボタンをクリックすると TypeError が出ます。（現状）
公式ドキュメントを確認し、イベントハンドラの書き方を
見直しましたが解決しませんでした。（試したこと）
コードを見ていただけますか？」
```

→ 相手がすぐに状況を把握でき、適切なアドバイスができる。

## 場面別の質問フォーマット

### 1. 技術的な問題を聞くとき

```
「[機能名]の実装で質問があります。

■ やりたいこと: [実現したいこと]
■ 現状: [発生している問題]
■ エラーメッセージ: [あれば]
■ 試したこと:
  - [試したこと1] → [結果]
  - [試したこと2] → [結果]
■ コード: [関連するコードを共有]

アドバイスをいただけますか？」
```

### 2. 仕様や要件を確認するとき

```
「[タスク名]の仕様について確認したいことがあります。

■ 自分の理解: [自分はこう理解している]
■ 疑問点: [ここがわからない/曖昧]
■ 確認したいこと: [Aという理解で合っているか？]

この認識で進めてよろしいでしょうか？」
```

### 3. やり方がわからないとき

```
「[作業名]の進め方がわからないので教えてください。

■ 状況: [何をしようとしているか]
■ 調べたこと: [調べた内容]
■ わからない点: [具体的にどこがわからないか]

どのように進めればよいでしょうか？」
```

### 4. 選択肢を相談するとき

```
「[課題]について、方針を相談させてください。

■ 課題: [解決したいこと]
■ 選択肢A: [方法Aの概要とメリット/デメリット]
■ 選択肢B: [方法Bの概要とメリット/デメリット]
■ 自分の考え: [自分はAがいいと思うが...]

どちらが適切でしょうか？」
```

## 対面で質問するときの流れ

### Step 1: 声をかける

```
「田中さん、今お時間よろしいですか？
[タスク名]について質問したいことがあるのですが」
```

### Step 2: 概要を伝える

```
「[機能名]の実装でエラーが出ていまして、
15分ほど調べたのですが解決できませんでした」
```

### Step 3: 詳細を説明

```
「やりたいことは[目的]で、
現状は[問題]という状態です。
[試したこと]を試しましたが、[結果]でした」
```

### Step 4: 質問する

```
「何か見落としている点があれば教えていただけますか？」
```

### Step 5: 教えてもらったら

```
「なるほど！[理解した内容]ということですね。
ありがとうございます。試してみます」
```

## やってはいけない質問の仕方

### NG 1: 丸投げ質問

```
× 「どうすればいいですか？」
× 「わかりません。教えてください」
× 「全然動きません」
```

→ 自分で何も考えていないように見える。

### NG 2: 質問の前置きが長い

```
× 「あの、えっと、先日のミーティングで話が出た件なんですけど、
    実は前から気になっていて、でもなかなか聞けなくて...（5分経過）
    ...で、質問なんですが...」
```

→ 結論を先に。前置きは最小限に。

### NG 3: 同じ質問を何度もする

```
× 先週も同じ質問をして教えてもらったのに、メモを取らずまた聞く
```

→ 教えてもらったことはメモを取る。

### NG 4: 相手の状況を無視

```
× 相手がミーティング中に話しかける
× 相手が集中して作業しているときに長時間拘束する
```

→ 「今、お時間よろしいですか？」と確認する。

## 教えてもらった後のフォロー

### 必ずやること

1. **メモを取る** -- 同じことを何度も聞かないように
2. **お礼を言う** -- 「ありがとうございます」
3. **結果を報告する** -- 「教えていただいた方法で解決しました」
4. **理解を確認する** -- 「つまり〇〇ということですね？」と復唱

### 結果報告の例

```
「田中さん、先ほどの件ですが、教えていただいた方法で
エラーが解消できました。ありがとうございました！」
```

> 結果報告をすると、先輩は「教えた甲斐があった」と感じてくれます。
