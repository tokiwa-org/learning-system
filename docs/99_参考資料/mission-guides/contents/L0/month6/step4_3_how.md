## 相談の3ステップ

```
Step 1: 状況を説明する
    ↓
Step 2: 自分の考えを伝える
    ↓
Step 3: 相手の意見を聞く
```

---

## Step 1: 状況を説明する

### 何を伝えるか

| 項目 | 内容 |
|------|------|
| 背景 | なぜこの相談が必要になったか |
| 現状 | 今どうなっているか |
| 問題 | 何が困っているか |

### 具体例

```
「プロフィール画面の画像アップロード機能を
実装しているのですが、（背景）
画像のファイルサイズ制限をどうするかで
判断に迷っています。（現状・問題）」
```

### ポイント

- **簡潔に** -- 長い経緯は不要。要点だけ伝える
- **事実ベースで** -- 感情ではなく事実を伝える
- **問題を明確に** -- 何に困っているか1文で言えるように

---

## Step 2: 自分の考えを伝える

### なぜ自分の考えが必要か

```
× 「どうすればいいですか？」
  → 丸投げ。相手が一から考える必要がある。

○ 「自分はAがいいと思いますが、いかがですか？」
  → 相手は「合っている/違う」を判断するだけでよい。
```

> 自分の考えを持つことは、成長のために不可欠です。

### 自分の考えの伝え方

```
「自分としては、[選択肢A]が良いと考えています。
理由は[根拠]です。
ただ、[懸念点]もあるため、ご意見をいただきたいです。」
```

### 考えがまとまらない場合

完璧な答えを出す必要はありません。

```
○ 「正直なところ、まだ判断がつきません。
    Aには[メリット]、Bには[メリット]があると思いますが、
    どちらを優先すべきか教えてください。」

○ 「自分なりに考えてみましたが、
    [ここ]の部分がまだ理解できていません。
    考え方のヒントをいただけますか？」
```

---

## Step 3: 相手の意見を聞く

### 聞くときの姿勢

1. **最後まで聞く** -- 途中で遮らない
2. **メモを取る** -- 大事なポイントを書き留める
3. **理解を確認する** -- 「つまり〇〇ということですね？」
4. **質問する** -- わからない点はその場で聞く

### 確認のフレーズ

```
「つまり、[自分の理解]ということですね？」
「確認ですが、[ポイント]という理解で合っていますか？」
「[用語]がわからなかったのですが、教えていただけますか？」
```

## 場面別の相談テンプレート

### 技術的な方針の相談

```
「[機能名]の実装方針について相談させてください。

■ 状況: [今の状況]
■ 選択肢:
  - A: [方法Aの概要]
  - B: [方法Bの概要]
■ 自分の考え: [自分はAがいいと思う。理由は...]
■ 懸念: [ここが気になっている]

ご意見をいただけますか？」
```

### 期限の相談

```
「[タスク名]の期限について相談させてください。

■ 状況: [想定より時間がかかっている]
■ 原因: [なぜ遅れているか]
■ 見込み: [完了見込み日]
■ 自分の考え: [対策案]

期限の調整は可能でしょうか？」
```

### 優先順位の相談

```
「タスクの優先順位について相談させてください。

■ 現在のタスク:
  1. [タスクA]（期限: 〇日）
  2. [タスクB]（期限: △日）
  3. [タスクC]（期限: □日）
■ 問題: すべてを期限内に終わらせるのが
  難しい状況です
■ 自分の考え: タスクAを最優先にして、
  タスクCの期限を延ばしたい

この優先順位で進めてよろしいでしょうか？」
```

### 人間関係の相談

```
「チームの件で相談させてください。

■ 状況: [困っていること]
■ 自分がやったこと: [自分なりの対処]
■ 相談したいこと: [アドバイスがほしいこと]

お時間があるときに、お話しさせていただけますか？」
```

## 相談後のフォロー

### 必ずやること

| アクション | タイミング | 例 |
|-----------|---------|-----|
| お礼 | 相談直後 | 「ありがとうございました」 |
| 結果報告 | 対応後 | 「教えていただいた方法でうまくいきました」 |
| 振り返り | 自分で | 学んだことをメモする |

### 結果報告の例

```
「田中さん、先日相談した画像アップロードの件ですが、
Base64方式で実装して、正常に動作しています。
ありがとうございました。」
```

> 結果を報告することで、次回も相談しやすい関係が作れます。

## まとめ

| ステップ | 内容 | ポイント |
|---------|------|---------|
| 1. 状況説明 | 背景・現状・問題を伝える | 簡潔に事実ベースで |
| 2. 自分の考え | 自分の意見を述べる | 完璧でなくてOK |
| 3. 意見を聞く | 相手の意見を聞く | メモを取り、理解を確認 |
| フォロー | お礼と結果報告 | 信頼関係を築く |
