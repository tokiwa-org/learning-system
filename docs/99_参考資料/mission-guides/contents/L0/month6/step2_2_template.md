## 基本テンプレート

### シンプル版（初心者向け）

```
【日報】YYYY年MM月DD日（曜日）

■ 今日やったこと
- [作業1]（かかった時間）
- [作業2]（かかった時間）

■ 明日やること
- [予定1]
- [予定2]

■ 困っていること・相談したいこと
- [あれば記載、なければ「特になし」]

■ 学んだこと・気づいたこと
- [あれば記載]
```

### 記入例

```
【日報】2025年4月7日（月）

■ 今日やったこと
- ログイン画面のHTML実装（2時間）
- CSSでのレイアウト調整（1.5時間）
- コードレビューの指摘対応（1時間）

■ 明日やること
- ログイン画面のバリデーション実装
- 単体テストの作成

■ 困っていること・相談したいこと
- CSSのFlexboxで意図通りにならない箇所がある
  → 明日の朝、田中さんに相談したい

■ 学んだこと・気づいたこと
- Flexboxのalign-itemsとjustify-contentの違いが理解できた
- コードレビューで変数名の付け方について学んだ
```

## 各項目の書き方

### 「今日やったこと」の書き方

**悪い例:**

```
- 作業した
- コードを書いた
- ミーティングに参加した
```

→ 何をしたのか全くわかりません。

**良い例:**

```
- ログイン画面のHTML/CSS実装（2時間）
  - フォーム部分のマークアップ完了
  - レスポンシブ対応は未着手
- チーム定例ミーティングに参加（1時間）
  - 来週のスプリント計画を確認
- TypeScript入門の第3章を読了（1時間）
```

→ 何を、どのくらいの時間で、どこまでやったかがわかります。

### ポイント

1. **具体的に書く** -- 「何の」作業かを明記
2. **時間を書く** -- 自分の時間管理にも役立つ
3. **進捗がわかるように** -- 完了/途中/未着手を明記

---

### 「明日やること」の書き方

**悪い例:**

```
- 続きをやる
- 頑張る
```

**良い例:**

```
- ログイン画面のバリデーション実装（見積もり: 3時間）
- 田中さんにFlexboxの件を相談（朝イチ）
- 単体テストを2件作成（見積もり: 2時間）
```

→ 翌朝すぐに作業を始められます。

---

### 「困っていること」の書き方

**悪い例:**

```
- わからないことがある
```

**良い例:**

```
- CSSのFlexboxでヘッダーのロゴとナビゲーションが
  横並びにならない
  - justify-content: space-between を試したが効かない
  - MDNのドキュメントを確認済み
  - 明日、田中さんに画面を見せて相談したい
```

→ 何に困っていて、何を試して、何をしたいかが明確です。

---

### 「学んだこと」の書き方

**悪い例:**

```
- いろいろ学んだ
```

**良い例:**

```
- git rebaseとgit mergeの違いを理解した
  - rebase: 履歴が一直線になる（きれい）
  - merge: マージコミットが作られる（履歴が残る）
- コードレビューで「早期リターン」パターンを教えてもらった
  - if文のネストを減らせるので可読性が上がる
```

→ 学びを言語化することで、知識が定着します。

## 日報の長さの目安

| レベル | 分量 | 状況 |
|--------|------|------|
| 最低限 | 5行程度 | 特に変化のない日 |
| 標準 | 10〜20行 | 通常の業務日 |
| 詳細 | 20行以上 | 問題が多い日、学びが多い日 |

> 短くても毎日書くことが大事です。「書かない」が最もNGです。
