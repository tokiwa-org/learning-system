## フィードバックとは

**フィードバック** とは、あなたの行動や成果物に対する評価・助言のことです。

### フィードバックの目的

| 目的 | 説明 |
|------|------|
| 品質の向上 | コードやドキュメントをより良くする |
| 成長の促進 | あなたのスキルを伸ばす |
| 認識の共有 | チームの基準を合わせる |
| 信頼の構築 | オープンなコミュニケーション |

> フィードバック = 「あなたへの攻撃」ではなく「成長のための助言」

## フィードバックを受けるときの心構え

### 4つの心構え

#### 1. 素直に受け入れる

```
× 「でも、こうした理由があって...」（言い訳から入る）
○ 「なるほど、そういう視点がありますね」（まず受け入れる）
```

#### 2. メモを取る

```
× 聞くだけで忘れる → 同じ指摘を繰り返し受ける
○ メモを取る → 次回から同じミスを防げる
```

#### 3. 感謝を伝える

```
× 黙ったまま
○ 「ご指摘ありがとうございます。次から気をつけます」
```

#### 4. 改善に活かす

```
× 指摘を聞いて終わり
○ 指摘の内容を実際に改善し、改善したことを報告する
```

## コードレビューでのフィードバック

### よくある指摘の種類

| 種類 | 例 | 受け止め方 |
|------|-----|-----------|
| バグの指摘 | 「この条件だとnullになりますよ」 | 見落としを教えてくれた |
| 可読性 | 「変数名をもっとわかりやすく」 | 読みやすいコードを学べる |
| 設計 | 「この処理は別の関数に分けましょう」 | 設計力が身につく |
| 規約 | 「チームのコーディング規約ではこう書きます」 | チームのルールを学べる |
| パフォーマンス | 「このクエリは重くなる可能性があります」 | 効率的なコードを学べる |

### コードレビューの受け方

```
1. 指摘を読む
    ↓
2. 内容を理解する（わからなければ質問する）
    ↓
3. 修正する
    ↓
4. 「修正しました。ご確認お願いします」と返信
```

### 指摘の意図がわからないとき

```
○ 「この指摘の意図を詳しく教えていただけますか？
    〇〇という理解であっていますか？」

× 黙って無視する
× わからないまま適当に修正する
```

## フィードバックの受け方 実践例

### 場面 1: 1on1でのフィードバック

```
上司: 「最近、報告のタイミングが少し遅いかな。
        問題が起きたらもっと早く教えてほしい」

良い受け方:
「ご指摘ありがとうございます。
確かに、自分で解決しようとして報告が遅れていました。
今後は問題に気づいた時点ですぐにご報告します。
具体的に、どのくらいの頻度で報告するのが理想ですか？」
```

### 場面 2: コードレビューでの指摘

```
レビュアー: 「この関数は1つのことだけやるようにしましょう。
              データの取得と表示の処理を分けてください」

良い受け方:
「ありがとうございます。確かに1つの関数に
2つの責務が混ざっていました。
データ取得用と表示用に分割して修正します。」
```

### 場面 3: 厳しいフィードバック

```
先輩: 「このコード、テストがまったくないね。
        テストなしで動いているように見えるだけだよ」

良い受け方:
「おっしゃる通りです。テストの重要性を軽く見ていました。
テストの書き方を教えていただけますか？
参考になるドキュメントがあれば共有いただきたいです。」
```

## フィードバックを成長に変える方法

### フィードバックノート

指摘された内容をノートに記録しましょう。

```
【フィードバックノート】

■ 日付: 2025/04/10
■ 誰から: 田中さん（コードレビュー）
■ 指摘内容: 変数名が抽象的すぎる
  - × data → ○ userProfileData
■ 学び: 変数名は「何のデータか」がわかるように命名する
■ 改善アクション: 今後は命名に注意してセルフチェックする

■ 日付: 2025/04/12
■ 誰から: 佐藤マネージャー（1on1）
■ 指摘内容: 報告のタイミングが遅い
■ 学び: 問題を感知したらすぐに報告する
■ 改善アクション: 問題発生時は15分以内に一報を入れる
```

### 定期的に振り返る

```
1週間ごと: 「今週指摘されたことは何か？改善できたか？」
1ヶ月ごと: 「先月と同じ指摘を受けていないか？」
3ヶ月ごと: 「指摘の内容がレベルアップしているか？」
```

> 同じ指摘が減っていれば、成長している証拠です。

## やってはいけないフィードバックの受け方

| NG行動 | なぜNGか |
|--------|---------|
| 言い訳から始める | 相手の意見を聞く姿勢がないように見える |
| 黙ってしまう | 理解しているのかわからない |
| 反論ばかりする | 改善する意思がないように見える |
| 落ち込みすぎる | フィードバックを避けるようになる |
| 無視する | 同じ指摘を繰り返し受けることになる |
