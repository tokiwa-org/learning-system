## 主要な日報ツール

### ツール比較

| ツール | 特徴 | よく使う現場 |
|--------|------|-------------|
| Slack | チャット形式、リアルタイム共有 | スタートアップ、Web系 |
| Microsoft Teams | Office連携、チャット + ドキュメント | 大企業、SIer |
| メール | フォーマルな報告 | 伝統的な企業 |
| Wiki（Confluence等） | ドキュメント型、検索しやすい | ドキュメント重視のチーム |
| 日報専用ツール | テンプレート完備、集計機能あり | 大規模組織 |

> どのツールを使うかはチームによって異なります。入社時に確認しましょう。

## Slackでの日報

### 基本的な投稿方法

```
投稿先: #daily-report チャンネル（チームによって異なる）

【日報】2025/04/07（月）山田太郎

*今日やったこと*
• ログイン画面のHTML実装（2h）
• チーム定例ミーティング（1h）

*明日やること*
• バリデーション実装

*困っていること*
• 特になし

*学び*
• Flexboxの使い方を理解した
```

### Slackの便利な機能

| 機能 | 使い方 | 用途 |
|------|--------|------|
| スレッド | 日報にコメントを返信 | フィードバックの受け取り |
| リアクション | 絵文字でリアクション | 「読んだよ」の意思表示 |
| ブックマーク | 重要な日報を保存 | 後から振り返り |
| 検索 | キーワードで日報を検索 | 過去の記録を参照 |

### Slackでの書式

```
*太字*    → 太字
_斜体_    → 斜体
`コード`  → インラインコード
• リスト  → 箇条書き
```

## Microsoft Teamsでの日報

### 基本的な投稿方法

```
投稿先: チームの「日報」チャネル

同じテンプレートをTeamsのチャネルに投稿します。
Teamsではマークダウン記法も一部使えます。
```

### Teamsの便利な機能

| 機能 | 用途 |
|------|------|
| チャネル | チーム全員が見られる |
| Wikiタブ | テンプレートを保存できる |
| Planner連携 | タスク管理と連動 |

## メールでの日報

### 基本的なフォーマット

```
件名: 【日報】2025/04/07 山田太郎

〇〇部長

お疲れ様です。山田です。
本日の日報をお送りします。

■ 本日の業務内容
1. ログイン画面のHTML実装（2時間）
2. チーム定例ミーティング（1時間）
3. TypeScript学習（1時間）

■ 明日の予定
1. バリデーション実装
2. 単体テスト作成

■ 所感
Flexboxの使い方がわかり、レイアウトが
効率的に組めるようになりました。

以上、よろしくお願いいたします。
山田太郎
```

### メール特有のポイント

- **件名にルールがある** ことが多い（日付と名前を入れる）
- **宛先** を間違えない
- **敬語** を使う（チャットより丁寧に）
- **返信不要** の場合はその旨を書く

## Wikiでの日報

### Confluence などのWikiツール

```
ページタイトル: 日報_20250407_山田太郎

テンプレート機能を使って統一フォーマットで作成。
検索性が高く、後から振り返りやすい。
```

### Wikiの利点

| 利点 | 説明 |
|------|------|
| 検索しやすい | キーワードで過去の日報を探せる |
| テンプレート | 毎回同じフォーマットで書ける |
| リンク | 関連するドキュメントと紐づけられる |
| 蓄積 | ナレッジとして蓄積できる |

## ツール選択のポイント

### チームのルールに従う

最も重要なのは、**チームが使っているツールで書く** ことです。

```
× 自分の好きなツールで書く
○ チームのルールに合わせる
```

### 入社時に確認すること

1. 日報はどのツールで書くか
2. どのチャンネル/フォルダに投稿するか
3. テンプレートはあるか
4. 提出期限はあるか（例: 18時まで）
5. 誰宛に書くか

## 作業メモの取り方

日報を楽に書くために、作業中にメモを取る習慣をつけましょう。

### メモの取り方

```
09:00 朝会参加
09:30 ログイン画面実装開始
10:15 Flexboxでハマる → MDN確認
10:45 解決、実装再開
12:00 昼休み
13:00 実装続き
14:00 コードレビュー依頼
14:30 レビュー指摘対応
15:30 TypeScript学習
17:00 日報作成 ← このメモをもとに書く
```

### メモツール

- **テキストエディタ**: VSCodeのメモファイル
- **付箋アプリ**: Windowsの付箋、Macのメモ
- **メモ帳**: 紙のメモ帳でもOK
