## チームミーティングの種類

### 主なミーティング一覧

| ミーティング | 頻度 | 目的 | 所要時間 |
|-------------|------|------|----------|
| デイリースタンドアップ | 毎日 | 進捗共有・問題の早期発見 | 15分 |
| スプリントレビュー | 1〜2週間ごと | 成果物のデモ・フィードバック | 1時間 |
| スプリントレトロスペクティブ | 1〜2週間ごと | 振り返り・改善 | 1時間 |
| 1on1 | 週1〜月1 | 上司との個別面談 | 30分〜1時間 |
| 全体会議 | 月1〜四半期ごと | 会社・部門の情報共有 | 1〜2時間 |

---

## デイリースタンドアップ（朝会）

### 目的

- チーム全員の進捗を短時間で共有する
- 問題や困りごとを早期に発見する
- その日の作業の方向性を確認する

### フォーマット（1人1〜2分）

```
1. 昨日やったこと
2. 今日やること
3. 困っていること（あれば）
```

### 発言例

```
「昨日はお知らせ一覧ページのHTML/CSSを実装しました。
今日はAPI連携を進めます。
特に困っていることはありません」
```

### 心構え

| やること | やらないこと |
|----------|-------------|
| 短く簡潔に話す | 詳細な技術議論をする |
| 困りごとは素直に伝える | 問題を隠す |
| 立って参加（オンラインは顔出し） | 座ってダラダラ話す |

---

## スプリントレビュー

### 目的

- スプリント（1〜2週間の開発期間）で完成した機能をデモする
- 関係者からフィードバックをもらう
- 次のスプリントの方向性を確認する

### 新人の役割

| 役割 | 内容 |
|------|------|
| 見学・学習 | 先輩のデモの仕方を学ぶ |
| メモを取る | フィードバック内容を記録する |
| 質問する | わからないことは質問OK |
| 担当機能のデモ | 慣れたら自分の作業をデモする |

### 心構え

- デモの準備は前日までに済ませる
- 動かない部分があれば事前に報告する
- フィードバックは素直に受け止める

---

## スプリントレトロスペクティブ（振り返り）

### 目的

- チームの働き方を振り返る
- 良かったこと・改善点を共有する
- 次のスプリントでの改善アクションを決める

### よくあるフォーマット

```
■ Keep（続けたいこと）
  - 毎日の朝会で進捗が見えやすかった
  - ペアプログラミングで学びが多かった

■ Problem（問題・課題）
  - 仕様の確認に時間がかかった
  - レビューの待ち時間が長かった

■ Try（試したいこと）
  - 仕様確認は朝一番に行う
  - レビュー依頼はSlackで通知する
```

### 新人の発言例

```
■ Keep
「先輩に丁寧に教えていただいて、理解が進みました」

■ Problem
「環境構築で詰まったとき、誰に聞けばいいかわかりませんでした」

■ Try
「困ったときの相談先リストを作っていただけると助かります」
```

## 1on1（ワンオンワン）

### 目的

- 上司と部下が定期的に1対1で話す場
- 業務の困りごと、キャリア、成長について話す
- 信頼関係を築く

### 話す内容の例

```
■ 業務について
  - 今の仕事で困っていること
  - 進捗の報告
  - 相談したいこと

■ 成長について
  - 学びたいこと
  - キャリアの希望
  - フィードバックのお願い

■ その他
  - 働き方の相談
  - チームの雰囲気について
```

### 心構え

| やること | 効果 |
|----------|------|
| 話したいことを事前にメモしておく | 有意義な時間になる |
| 素直に困りごとを話す | 問題が早く解決する |
| フィードバックを求める | 成長につながる |
| 感謝を伝える | 良い関係が築ける |

### 1on1で避けるべきこと

- 何も話すことがない、と言う
- 問題を隠す
- 愚痴だけで終わる

## 全体会議

### 目的

- 会社や部門の方針・情報を共有する
- 重要なアナウンスを行う
- 全員が同じ認識を持つ

### 新人の心構え

| やること | 理由 |
|----------|------|
| メモを取る | 後で見返せるように |
| 略語や専門用語をメモする | 後で調べる |
| 質疑応答で質問する | 理解を深める |
| 出席は必須 | 情報についていく |

---

## ミーティング共通の心構え

### 準備

| 準備項目 | 内容 |
|----------|------|
| 時間を確認 | 開始5分前には準備完了 |
| 資料を確認 | 事前共有された資料を読んでおく |
| 発言を準備 | 自分が話すことを整理しておく |
| 環境を整える | オンラインならカメラ・マイクをチェック |

### 参加中

| やること | やらないこと |
|----------|-------------|
| 集中して聞く | 内職する（別の作業をする） |
| メモを取る | ぼんやりする |
| 適度に反応する（うなずく等） | 無反応でいる |
| 質問・発言する | 黙り続ける |

### 終了後

| やること | 効果 |
|----------|------|
| メモを整理する | 後で見返せる |
| アクションアイテムを確認する | やるべきことを明確に |
| 不明点を質問する | 理解を深める |
