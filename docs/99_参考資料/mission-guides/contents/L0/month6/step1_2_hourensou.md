## 報連相とは

**報連相（ほうれんそう）** は、日本のビジネスシーンで最も重要なコミュニケーションの基本です。

| 種類 | 読み方 | 一言で言うと |
|------|--------|-------------|
| **報** 告 | ほうこく | 結果を伝える |
| **連** 絡 | れんらく | 事実を共有する |
| **相** 談 | そうだん | 判断を仰ぐ |

---

## 報告（ほうこく）

### 定義

**指示された仕事の結果や経過を、指示した人に伝えること。**

### ポイント

- 指示を出した人に対して行う（自分 → 上司・先輩）
- 完了報告だけでなく、途中経過も含む
- 良い結果も悪い結果も伝える

### 具体例

| 場面 | 報告の例 |
|------|---------|
| タスク完了時 | 「依頼されたバグ修正が完了しました」 |
| 途中経過 | 「現在50%まで進んでいます。予定通り水曜に完了見込みです」 |
| 問題発生時 | 「テスト中にエラーが見つかりました。調査中です」 |
| 遅延時 | 「申し訳ありません。想定より時間がかかり、1日遅れそうです」 |

### 悪い報告の例

```
× 「終わりました」 → 何が終わったのかわからない
× 報告しない      → 上司は進捗がわからず不安
× 完了時のみ報告  → 問題が手遅れになる可能性
```

### 良い報告のテンプレート

```
【報告】
■ 件名: [何についての報告か]
■ 状況: [完了 / 進行中 / 問題発生]
■ 詳細: [具体的な内容]
■ 次のアクション: [次に何をするか]
```

## 連絡（れんらく）

### 定義

**関係者に事実や情報を共有すること。**

### ポイント

- 上下関係に関係なく、関係者全員に行う
- 自分の意見は含めず、事実のみを伝える
- タイムリーに共有する

### 具体例

| 場面 | 連絡の例 |
|------|---------|
| 予定変更 | 「明日のミーティングは14時から15時に変更になりました」 |
| 共有事項 | 「開発環境のURLが変わりました。新URLは〇〇です」 |
| 不在連絡 | 「明日は午前中に通院のため、午後からの出社になります」 |
| 情報共有 | 「お客様から仕様変更の要望がありました」 |

### 報告との違い

| 比較項目 | 報告 | 連絡 |
|---------|------|------|
| 方向 | 部下 → 上司 | 誰 → 誰でも |
| 内容 | 結果や経過 | 事実や情報 |
| きっかけ | 指示された仕事について | 関係者が知るべき情報 |

---

## 相談（そうだん）

### 定義

**判断に迷ったときや困ったときに、意見やアドバイスを求めること。**

### ポイント

- 自分の考えを持った上で相談する
- 早めに相談する（問題が大きくなる前に）
- 相談相手の時間を尊重する

### 具体例

| 場面 | 相談の例 |
|------|---------|
| 判断に迷う | 「AとBの方法があるのですが、どちらが良いでしょうか？」 |
| 技術的問題 | 「このエラーが解決できません。アドバイスいただけますか？」 |
| 優先順位 | 「タスクが3つあるのですが、どれから着手すべきですか？」 |
| 期限の相談 | 「この量だと期限に間に合わなそうです。相談させてください」 |

### 良い相談のテンプレート

```
【相談】
■ 相談したいこと: [何について相談したいか]
■ 現状: [今どうなっているか]
■ 自分の考え: [自分はこう思うが...]
■ 聞きたいこと: [何を教えてほしいか]
```

## 報連相の使い分け早見表

| こんなとき | 報連相 | 例 |
|-----------|--------|-----|
| タスクが完了した | **報告** | 「完了しました」 |
| 予定が変わった | **連絡** | 「時間が変更です」 |
| やり方がわからない | **相談** | 「教えてください」 |
| エラーが発生した | **報告** | 「問題が起きました」 |
| 会議室が変わった | **連絡** | 「場所が変わります」 |
| どちらを選ぶべきか | **相談** | 「意見を聞かせてください」 |
| 遅刻する | **連絡** | 「遅れます」 |
| 進捗を伝える | **報告** | 「50%完了です」 |

## よくある失敗パターン

### 1. 報告が遅い

```
悪い例:
月曜: タスク開始
金曜: 「すみません、間に合いませんでした...」

良い例:
月曜: タスク開始
水曜: 「想定より難しく、遅れそうです」→ 対策を相談
木曜: 「〇〇の方法で進めています。金曜に完了予定です」
```

### 2. 連絡をしない

```
悪い例:
（電車が遅延しているが何も言わない）
→ チームが心配する

良い例:
「電車遅延のため15分ほど遅れます。申し訳ありません」
→ チームが予定を調整できる
```

### 3. 相談せずに一人で抱え込む

```
悪い例:
3日間一人で悩み続ける → 結局解決できない

良い例:
15分調べてわからない → 先輩に相談 → 5分で解決
```

| 共通の鉄則 | 早めに、正確に、こまめに |
