## ミッション概要

社内ポータルサイト リニューアルの最初の1週間（2/2〜2/6）の作業計画を作成してください。

### 前提情報

```
プロジェクト: 社内ポータルサイト リニューアル
今週の範囲: 2026年2月2日（月）〜 2月6日（金）
最終締め切り: 2月13日（金）
1日の作業可能時間: 6時間（会議・休憩除く）
今週の固定予定:
  - 月曜 10:00-11:00 チーム定例ミーティング
  - 水曜 14:00-15:00 デザインレビュー（佐藤さんと）
  - 金曜 16:00-17:00 週次進捗報告会
```

### 今週のタスク

| # | タスク | 見積もり時間 | 依存関係 |
|---|--------|-------------|----------|
| 1 | トップページのHTML/CSS実装 | 8時間 | デザインデータ（月曜朝に受領予定） |
| 2 | ナビゲーションメニュー作成 | 4時間 | なし |
| 3 | お知らせ一覧の画面だけ先に作成 | 3時間 | #1のレイアウトを参考にする |

**注意**: #4（レスポンシブ）と #5（テスト）は来週の作業です。

### 達成条件

- [ ] 1週間の日次計画を作成した
- [ ] マイルストーンを設定した
- [ ] 見積もりにバッファを含めた
- [ ] トラブル発生時の対応策を考えた

## Mission 3: トラブル対応を考えよう

以下の3つのトラブルが発生した場合の対応を考えてください。

### トラブル1: デザインデータが月曜の午後になっても届かない

<details>
<summary>解答例</summary>

```
【対応策】
1. 午前中に佐藤さんにチャットで状況を確認する
2. デザインデータなしでも進められる作業を先に行う
   - ナビゲーションのHTML/CSS（デザインに依存しない部分）
   - お知らせ一覧ページのHTML骨組み
3. 火曜日の午前中にもデータが届かない場合:
   - 上司に報告し、佐藤さんの状況を確認してもらう
   - ワイヤーフレームベースで仮実装を進める
4. 影響の報告:
   「デザインデータの受領が1日遅れたため、
    トップページのCSS実装が1日遅れる見込みです。
    金曜のバッファで吸収可能と考えています」
```

</details>

---

### トラブル2: 水曜のレビューで大幅な修正指示が出た

<details>
<summary>解答例</summary>

```
【対応策】
1. 修正内容を具体的にリストアップする
2. 各修正の見積もり時間を出す
3. 影響範囲を分析する:
   - 小規模修正（色やフォントの変更）→ バッファ内で対応可能
   - 大規模修正（レイアウト変更）→ 計画の見直しが必要
4. 大規模修正の場合の対応案:
   案1: お知らせ一覧ページの着手を来週に延期し、修正を優先
   案2: 修正を優先度で分け、重要なものだけ今週対応
   案3: 一部の修正を来週に回す（佐藤さんと相談）
5. 上司に報告:
   「デザインレビューで修正指示が出ました。
    修正に約○時間かかる見込みです。
    対策として案1〜3を考えています。ご相談させてください」
```

</details>

---

### トラブル3: 木曜に体調不良で午後から早退することになった

<details>
<summary>解答例</summary>

```
【対応策】
1. 早退前にやること:
   - 現在の進捗状況をカンバンボードに更新する
   - 作業中のファイルを保存・コミットする
   - 上司に残りタスクと進捗を報告する
2. 金曜日の計画を見直す:
   - 木曜午後の作業を金曜午前に移す
   - 金曜のバッファ時間を作業に充てる
   - 優先度の低い作業は来週に延期
3. 金曜でも間に合わない場合:
   - 上司に早めに相談
   - 先輩に一部タスクの支援を依頼
4. 体調回復を最優先にする（無理をしない）
```

</details>
