## 計画の3つのレベル

### 全体像

| レベル | スコープ | 内容 |
|--------|----------|------|
| **マイルストーン** | プロジェクト全体 | 大きな節目の目標と期限 |
| **週次計画** | 今週の作業 | 今週やることのリスト |
| **日次計画** | 今日の作業 | 今日やることと時間割 |

```
プロジェクト全体
├── マイルストーン1（2/5）
│   ├── 週次計画（2/2-2/6）
│   │   ├── 月曜日の日次計画
│   │   ├── 火曜日の日次計画
│   │   └── ...
│   └── ...
├── マイルストーン2（2/10）
└── 最終締め切り（2/13）
```

## ステップ1: ゴールから逆算する

### 逆算思考とは

**最終ゴールから遡って、いつまでに何を終わらせるかを決める方法**

```
通常の考え方（前から）:
月曜 → 火曜 → 水曜 → ... → いつ終わる？

逆算思考（後ろから）:
金曜に完了 ← 木曜に確認 ← 水曜に作成 ← 火曜に準備 ← 月曜に情報収集
```

### 逆算の手順

1. 最終締め切りを確認する
2. 最終日の1日前を「完了目標日」にする
3. 完了目標日から逆算して、各作業のスケジュールを決める

### 具体例

```
タスク: 月次報告書の作成
最終締め切り: 金曜日

金曜日: 最終確認 → 提出
木曜日: 上司レビュー → 修正
水曜日: 報告書の本文作成
火曜日: データ集計・グラフ作成
月曜日: 情報収集・構成検討
```

## ステップ2: マイルストーンを設定する

### マイルストーンとは

**プロジェクトの中間地点に置く、チェックポイント**

道路の「マイル標石（マイルストーン）」が語源です。

### マイルストーンの設定方法

| ルール | 説明 |
|--------|------|
| **具体的な成果物を定義する** | 「〜が完成している」状態を明記 |
| **1週間に1つ以上置く** | 長すぎると進捗が見えない |
| **完了の判定が明確** | 誰が見ても完了かどうか判断できる |

### 具体例

```
プロジェクト: 社内ポータルサイト リニューアル（2週間）

マイルストーン1（2/5 木）:
  □ トップページのHTMLが完成している
  □ ナビゲーションメニューが動作する

マイルストーン2（2/9 月）:
  □ お知らせ一覧ページが完成している
  □ APIからデータが表示されている

マイルストーン3（2/11 水）:
  □ 全ページのレスポンシブ対応が完了している

最終マイルストーン（2/13 金）:
  □ 全ページのテストが完了している
  □ バグがすべて修正されている
```

## ステップ3: 週次計画を作る

### 週の初めに作るToDoリスト

```
今週やること（2/2〜2/6）:

□ [月] デザインデータを佐藤さんから受け取る
□ [月] トップページのHTML骨組みを作成する
□ [火] トップページのCSS実装を進める
□ [火] ナビゲーションのHTML作成
□ [水] トップページのCSS実装を完了する
□ [水] ナビゲーションのCSS実装
□ [木] ナビゲーションの動作確認
□ [木] デザインレビューを佐藤さんに依頼
□ [金] レビュー対応・修正
□ [金] ★マイルストーン1の達成確認
```

### 週次計画のコツ

| コツ | 説明 |
|------|------|
| 月曜〜木曜に作業を集中 | 金曜はバッファとレビュー対応に使う |
| 1日の作業量は6時間で計算 | 会議・休憩・割り込みで2時間は消える |
| マイルストーンを金曜に置く | 週末前に達成状況を確認できる |

## ステップ4: 日次計画を作る

### 毎朝5分で作る「今日やることリスト」

```
今日やること（2/3 火曜日）:

09:00-09:30  メール・チャットの確認
09:30-12:00  トップページのCSS実装
12:00-13:00  昼休み
13:00-14:00  チーム定例ミーティング
14:00-16:00  ナビゲーションのHTML作成
16:00-16:30  休憩
16:30-17:30  ナビゲーションのCSS着手
17:30-18:00  今日の振り返り・明日の準備
```

### 日次計画のポイント

| ポイント | 説明 |
|----------|------|
| **時間を区切る** | 「午前中に〜」ではなく「9:30-12:00に〜」 |
| **集中作業は午前に** | 午前は集中力が高い時間帯 |
| **会議後は軽い作業を** | 会議後は集中力が下がりやすい |
| **振り返り時間を確保** | 夕方15〜30分で今日の進捗を確認 |

## 計画の見直し

### 計画は変わるもの

計画通りに100%進むことはまれです。定期的に見直しましょう。

| タイミング | やること |
|-----------|----------|
| 毎日（夕方） | 今日の進捗を確認し、明日の計画を調整 |
| 週の半ば（水曜） | 週次計画の進捗を確認し、残りの日程を調整 |
| マイルストーン時 | 達成状況を確認し、以降の計画を見直す |

### 計画変更の判断基準

```
計画変更が必要なとき:
├── 予定の50%以上が未完了
├── 新しいタスクが追加された
├── 依存先の遅れが発生した
└── 想定以上に時間がかかっている

→ 上司や先輩に早めに相談する
```
