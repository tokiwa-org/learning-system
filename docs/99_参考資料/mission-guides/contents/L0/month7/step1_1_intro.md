## 頭の中で管理する限界

### こんな経験はありませんか？

- やることがたくさんあって、何から始めればいいかわからない
- 「あ、あれもやらなきゃ」と思い出して焦る
- 頼まれた作業を忘れてしまった
- 締め切りギリギリになって慌てた

これらはすべて **タスク管理ができていない** ことが原因です。

### 人間の記憶の限界

| 項目 | 数値 |
|------|------|
| 短期記憶で保持できる情報 | 約7個（マジックナンバー7） |
| 同時に意識できるタスク | 2〜3個が限界 |
| 記憶の信頼性 | 時間とともに急速に低下 |

> 先輩「頭の中だけで管理しようとすると、必ず漏れが出る。だからプロは全部ツールに記録するんだ」

---

## プロのタスク管理

### プロとアマチュアの違い

| 項目 | アマチュア | プロ |
|------|-----------|------|
| タスクの記録 | 頭の中・メモ書き | ツールで一元管理 |
| 優先順位 | なんとなく | 明確な基準で判断 |
| 進捗の把握 | 聞かれたら確認 | 常に可視化 |
| 遅延への対応 | ギリギリで報告 | 早期に察知して報告 |
| 完了の定義 | なんとなく終わり | 完了条件が明確 |

### なぜツールで管理するのか

1. **忘れない**: すべてのタスクが記録されている
2. **見える化**: 全体の状況が一目でわかる
3. **共有できる**: チームメンバーに状況を共有できる
4. **振り返れる**: 過去の作業履歴を確認できる

## タスク管理で身につく力

### 仕事の基礎力

タスク管理は、あらゆる仕事の **土台** となるスキルです。

```
タスク管理ができると...
├── 信頼される → 「この人に任せれば安心」
├── 効率が上がる → 無駄な作業が減る
├── 余裕が生まれる → 焦らずに仕事ができる
└── 成長が加速する → 振り返りから学べる
```

### 業界での重要性

| 職種 | タスク管理の活用例 |
|------|-------------------|
| エンジニア | GitHub Issues でバグ管理、スプリント計画 |
| デザイナー | Trello でデザインタスクの進行管理 |
| マーケター | Notion でキャンペーン施策の管理 |
| マネージャー | Jira でプロジェクト全体の進捗管理 |

> 先輩「どんな職種でもタスク管理は必須だよ。早いうちに身につけておくと、一生役に立つ」
