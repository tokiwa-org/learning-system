## 代表的なタスク管理ツール

### ツール一覧

| ツール名 | 開発元 | 特徴 | よく使う現場 |
|----------|--------|------|-------------|
| **GitHub Issues** | GitHub | コードと連携、無料 | エンジニアチーム |
| **Jira** | Atlassian | 高機能、大規模向け | 企業のIT部門 |
| **Trello** | Atlassian | ビジュアルで直感的 | 小〜中規模チーム |

| **Backlog** | Nulab | 日本製、日本語に強い | 日本企業 |

## 各ツールの詳細

### GitHub Issues

```
特徴:
├── ソースコードと直接連携できる
├── Pull Request と紐づけられる
├── ラベルやマイルストーンで分類
├── 無料で使える（パブリックリポジトリ）
└── エンジニアなら必ず触れるツール
```

**向いている場面**: ソフトウェア開発プロジェクト

### Jira

```
特徴:
├── エピック → ストーリー → タスクの階層管理
├── スプリント管理（アジャイル開発向け）
├── 詳細なレポート・分析機能
├── カスタマイズ性が非常に高い
└── 大企業での導入実績が豊富
```

**向いている場面**: 大規模プロジェクト、アジャイル開発

### Trello

```
特徴:
├── ドラッグ＆ドロップで直感操作
├── カード形式でタスクを管理
├── 見た目がシンプルでわかりやすい
├── 無料プランでも十分使える
└── 非エンジニアにも親しみやすい
```

**向いている場面**: 小規模チーム、個人のタスク管理

### Notion

```
特徴:
├── ドキュメント・Wiki・タスクを一元管理
├── データベース機能が柔軟
├── テンプレートが豊富
├── カスタマイズ自由度が高い
└── チーム全体のナレッジ管理にも使える
```

**向いている場面**: ドキュメントとタスクを統合管理したいチーム

### Backlog

```
特徴:
├── 日本語UIが自然で使いやすい
├── プロジェクト管理 + Git + Wiki が一体
├── ガントチャート標準搭載
├── 日本企業向けのサポート体制
└── 請求書払いなど日本のビジネス慣行に対応
```

**向いている場面**: 日本企業のプロジェクト管理

## ツール選びのポイント

### 比較表

| 観点 | GitHub Issues | Jira | Trello | Notion | Backlog |
|------|:---:|:---:|:---:|:---:|:---:|
| 学習コスト | 低 | 高 | 低 | 中 | 中 |
| カスタマイズ性 | 中 | 高 | 低 | 高 | 中 |
| コード連携 | 強 | 強 | 弱 | 弱 | 中 |
| 無料プラン | あり | あり | あり | あり | なし |
| 日本語サポート | 部分的 | あり | あり | あり | 強い |

### どれを使えばいいの？

> 先輩「正直、チームで決まったツールを使うことがほとんど。大事なのは **ツールの使い方** じゃなくて **タスク管理の考え方** だよ」

**結論**: どのツールでも以下の基本機能は共通です。

---

## 全ツール共通の基本機能

### 1. タスクの作成

- タスクを1つずつ「チケット」として登録する
- タイトル、説明、担当者、期限を設定する

### 2. ステータス管理

- タスクの状態（未着手 / 作業中 / 完了）を管理する
- 状態の変更が記録される

### 3. 分類・フィルタ

- ラベルやカテゴリでタスクを分類する
- 自分のタスクだけを表示する

### 4. コミュニケーション

- タスクにコメントを残せる
- 関係者にメンション（通知）を送れる
