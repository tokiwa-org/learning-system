## 議事録とは

議事録（ぎじろく）とは、会議で話し合われた内容、決定事項、次のアクションを記録した文書です。

### 議事録の目的

| 目的 | 説明 |
|------|------|
| **決定事項の記録** | 何が決まったかを明確にする |
| **認識の統一** | 参加者全員が同じ理解を持つ |
| **アクションの明確化** | 誰が何をいつまでにやるかを記録する |
| **不参加者への共有** | 会議に出られなかった人に内容を伝える |
| **証跡として残す** | 後から「あの時こう決まった」を確認できる |

---

## 議事録がないとどうなるか

### シナリオ: 議事録なしの会議

```
月曜日の会議:
  「リリース日は来週の金曜日にしよう」
  「テスト担当は佐藤さんね」

水曜日:
  田中「リリースは再来週って話じゃなかったっけ？」
  佐藤「え？テスト担当って僕でしたっけ？」
  鈴木「会議に出てなかったので何も分かりません...」

  → 何も決まっていないのと同じ状態
```

### シナリオ: 議事録ありの会議

```
月曜日の会議後、議事録を共有:
  決定事項:
    - リリース日: 来週金曜日（4/11）
    - テスト担当: 佐藤さん

水曜日:
  田中「あ、議事録を確認...来週金曜日か」
  佐藤「議事録に書いてあるね。テストケース作成を始めよう」
  鈴木「議事録を読んで内容を把握しました」

  → 全員が正確に状況を把握
```

## 議事録に含めるべき内容

### 必須項目

| 項目 | 内容 | 例 |
|------|------|-----|
| **日時** | 会議の日付と時間 | 2025-04-01 10:00-11:00 |
| **場所** | 会議の場所やオンラインツール | 会議室A / Zoom |
| **参加者** | 出席者の名前 | 田中、佐藤、鈴木 |
| **アジェンダ** | 議題（話し合うテーマ） | 1. 進捗確認 2. 仕様決定 |
| **議論内容** | 話し合った内容の要約 | AとBの方式を比較検討した |
| **決定事項** | 決まったこと | OAuth2.0を採用する |
| **アクションアイテム** | 次にやること（担当者・期限付き） | 田中: 実装（4/7まで） |
| **次回予定** | 次の会議の日時 | 次回: 4/8 10:00 |

---

### 書かなくてよいこと

| 書かなくてよいもの | 理由 |
|-------------------|------|
| 全員の発言をそのまま | 要約で十分。重要な発言のみ記録 |
| 雑談の内容 | 業務に関係ない話は省略 |
| 個人的な感想 | 客観的な事実のみ記録 |

## 良い議事録と悪い議事録

### 悪い議事録の例

```markdown
今日の会議
いろいろ話し合った。
認証の話とか。
佐藤さんが何かやるらしい。
```

問題点：
- いつの会議か分からない
- 何が決まったか分からない
- 「何か」では具体的に何をするか分からない
- 期限がない

### 良い議事録の例

```markdown
# 定例ミーティング 議事録
日時: 2025-04-01 10:00-11:00
参加者: 田中、佐藤、鈴木

## 決定事項
- 認証方式はOAuth2.0を採用する
- リリース日は4月15日とする

## アクションアイテム
- [ ] 佐藤: OAuth2.0のライブラリ選定（4/4まで）
- [ ] 田中: テスト計画書の作成（4/7まで）

## 次回
4月8日(火) 10:00 会議室A
```

| ポイント | 内容 |
|----------|------|
| 議事録とは | 会議の決定事項とアクションを記録する文書 |

| 目的 | 認識統一、アクション明確化、証跡 |
| コツ | 全発言の書き起こしではなく、要点をまとめる |
