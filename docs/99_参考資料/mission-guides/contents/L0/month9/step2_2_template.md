## 標準テンプレート

以下は、多くの現場で使われている議事録の標準テンプレートです。

```markdown
# [会議名] 議事録

## 基本情報

| 項目 | 内容 |
|------|------|
| 日時 | YYYY-MM-DD HH:MM - HH:MM |
| 場所 | （会議室名 / オンラインツール名） |
| 参加者 | （名前をカンマ区切り） |
| 記録者 | （あなたの名前） |

## アジェンダ

1. （議題1）
2. （議題2）
3. （議題3）

## 議論内容

### 議題1: （タイトル）

- （議論のポイントを箇条書き）
- （重要な発言や意見）

### 議題2: （タイトル）

- （議論のポイントを箇条書き）

## 決定事項

- （決まったこと1）
- （決まったこと2）

## アクションアイテム

| 担当 | 内容 | 期限 |
|------|------|------|
| （名前） | （やること） | （YYYY-MM-DD） |
| （名前） | （やること） | （YYYY-MM-DD） |

## 次回予定

- 日時: YYYY-MM-DD HH:MM
- 場所: （会議室名 / オンラインツール名）
- アジェンダ（予定）:
  1. （次回の議題）
```

## テンプレートの使い方

### ステップ1: 会議前に準備する

会議が始まる前に、分かっている情報を埋めておきます。

```markdown
# 開発定例ミーティング 議事録

## 基本情報

| 項目 | 内容 |
|------|------|
| 日時 | 2025-04-01 10:00 - 11:00 |
| 場所 | Zoom |
| 参加者 | （会議開始後に記入） |
| 記録者 | 山田太郎 |

## アジェンダ

1. 先週の進捗報告
2. 新機能の仕様検討
3. リリーススケジュールの確認
```

> 事前にアジェンダが共有されていれば、それを記入しておきましょう。

---

### ステップ2: 会議中にメモを取る

完璧な文章でなくてOK。キーワードや要点だけメモします。

```markdown
## 議論内容

### 議題1: 先週の進捗報告

- 田中: ログイン画面完了、テスト中
- 佐藤: API設計中、仕様で質問あり → 別途MTGで確認
- 鈴木: インフラ構築 80%完了

### 議題2: 新機能の仕様検討

- 認証方式 → OAuth2.0 vs 独自認証
- セキュリティ面で OAuth2.0が有利という意見が多数
- 工数は OAuth2.0 の方が少ない
```

---

### ステップ3: 会議後に清書する

メモを整理して、決定事項とアクションアイテムを明確にします。

```markdown
## 決定事項

- 認証方式はOAuth2.0を採用する
- リリース日は4月15日（火）とする
- APIの仕様に関する質問は4/3のMTGで解決する

## アクションアイテム

| 担当 | 内容 | 期限 |
|------|------|------|
| 田中 | ログイン機能のテスト完了 | 2025-04-04 |
| 佐藤 | OAuth2.0のライブラリ選定 | 2025-04-04 |
| 鈴木 | ステージング環境の構築 | 2025-04-07 |

## 次回予定

- 日時: 2025-04-08 10:00
- 場所: Zoom
- アジェンダ（予定）:
  1. 進捗確認
  2. API仕様のレビュー
```

## 場面別テンプレートのバリエーション

### 短い打ち合わせ用（15-30分）

```markdown
# [打ち合わせ名] メモ
日時: YYYY-MM-DD HH:MM
参加者: A, B

## 内容
- （話した内容）

## 決定事項
- （決まったこと）

## TODO
- [ ] A: （やること）（期限）
- [ ] B: （やること）（期限）
```

### 1on1ミーティング用

```markdown
# 1on1 メモ
日時: YYYY-MM-DD
参加者: （上司名）、（自分の名前）

## 報告事項
- （進捗や成果）

## 相談事項
- （相談したいこと）

## フィードバック
- （もらったアドバイス）

## ネクストアクション
- [ ] （次にやること）
```

## テンプレートを保存しておこう

よく使うテンプレートは、以下のような場所に保存しておくと便利です。

| 保存場所 | 特徴 |
|----------|------|
| GitHubリポジトリ | チームで共有できる |
| Notion / Confluence | テンプレート機能がある |
| ローカルフォルダ | すぐにコピーして使える |
| テキストエディタのスニペット | ショートカットで呼び出せる |
