## ストーリー

> 「毎週金曜日に週報を出してもらってるんだけど、来週からお願いできる？」
>
> 「週報ですか。何を書けばいいんですか？」
>
> 「今週やったこと、来週やること、困ってることの3つだよ。テンプレートを渡すね」

---

## 週報の目的

| 目的 | 説明 |
|------|------|
| **進捗共有** | 今週の成果をチームに共有する |
| **計画の明示** | 来週やることを宣言する |
| **課題の共有** | 困っていることを早めに伝える |
| **振り返り** | 1週間の仕事を振り返る |

---

## 週報テンプレート

```markdown
# 週報 YYYY-MM-DD 〜 YYYY-MM-DD

## 報告者
（あなたの名前）

## 今週の成果

### 完了したタスク
- （タスク1）
- （タスク2）

### 進行中のタスク
- （タスク3）進捗: XX%
- （タスク4）進捗: XX%

## 来週の計画
- （来週やること1）
- （来週やること2）

## 課題・リスク
- （課題や困っていること）
- （プロジェクトのリスク）

## 学んだこと・気づき
- （今週学んだ技術や気づき）
```

---

## 各セクションの書き方

### 今週の成果

**やったことを具体的に書く**のがポイントです。

### 悪い例

```markdown
## 今週の成果
- コーディングした
- テストした
- ミーティングに参加した
```

### 良い例

```markdown
## 今週の成果

### 完了したタスク
- ユーザー登録APIの実装を完了（POST /api/users）
- バリデーションのユニットテスト20件を作成し全件パス
- コードレビューで指摘された3件を修正

### 進行中のタスク
- ログイン機能の実装: 70%完了（認証ロジックまで完了、画面未着手）
- E2Eテストの準備: 40%完了（テスト環境の構築中）
```

> ポイント: 何を作ったか、何件テストしたかなど、数字で具体的に書く。

---

### 来週の計画

来週やることを **優先度順** に書きます。

```markdown
## 来週の計画

1. ログイン機能の実装完了（優先度: 高）
2. E2Eテスト環境の構築完了（優先度: 高）
3. パスワードリセット機能の設計着手（優先度: 中）
```

> ポイント: 優先度を付けると、上司やチームが確認しやすくなります。

---

### 課題・リスク

**困っていること** や **プロジェクトに影響するリスク** を正直に書きます。

```markdown
## 課題・リスク

- **API仕様書が未確定**
  - 影響: バックエンド連携の実装が着手できない
  - 対応: 佐藤さんに4/10までに確定を依頼済み

- **テスト環境のディスク容量不足**
  - 影響: E2Eテストが実行できない可能性
  - 対応: インフラチームに増設を依頼中
```

> ポイント: 課題には「影響」と「対応（自分がどうしているか）」をセットで書く。

---

### 学んだこと・気づき

このセクションは必須ではありませんが、書くことで以下のメリットがあります。

- 自分の成長を記録できる
- チームメンバーにも知見を共有できる
- 上司が育成の参考にできる

```markdown
## 学んだこと・気づき

- JWTのリフレッシュトークンの仕組みを理解した
  - アクセストークンは短期間（15分）、リフレッシュトークンは長期間（7日）
- テストを先に書くとバグが早期に見つかることを実感した
```

## 週報の具体例（完成版）

```markdown
# 週報 2025-04-01 〜 2025-04-05

## 報告者
山田太郎

## 今週の成果

### 完了したタスク
- ユーザー登録API（POST /api/users）の実装とテスト完了
- バリデーションのユニットテスト20件作成（全件パス）
- コードレビュー指摘事項3件の修正完了

### 進行中のタスク
- ログイン機能の実装: 70%（認証ロジック完了、画面は未着手）
- E2Eテスト環境の構築: 40%（Docker環境の設定中）

## 来週の計画

1. ログイン機能の実装完了（優先度: 高）
2. E2Eテスト環境の構築完了（優先度: 高）
3. パスワードリセット機能の設計着手（優先度: 中）

## 課題・リスク

- **API仕様書が未確定**（認証関連）
  - 影響: 一部のエンドポイント実装が着手できない
  - 対応: 佐藤さんに4/10までに確定を依頼済み

## 学んだこと・気づき

- JWT認証のリフレッシュトークンの仕組みを理解した
- Docker Composeでテスト環境を構築する方法を学んだ
```

## 週報を書くタイミングと習慣

| タイミング | やること |
|-----------|---------|
| **毎日（帰る前に1分）** | 今日やったことを箇条書きで3行メモする |
| **金曜日の午後** | 1週間分のメモをまとめて週報を作成する |
| **金曜日の夕方** | チームに共有する |

> 毎日1分メモを取る習慣をつけると、金曜日の週報作成がとても楽になります。
