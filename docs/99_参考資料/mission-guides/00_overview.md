# ミッション型カリキュラム設計ガイド - 概要

## 目的

本ガイドは、職級（L0-L5）別のミッション型カリキュラム設計の標準を定義します。
月20時間の学習時間を前提に、**ロードマップのスキル定義と連動**した学習内容と難易度を設計します。

## 2. ロードマップとの連動

### 2.1 ロードマップ構造

Neo4jに定義された82件のスキル項目（RoadmapItem）を基に、各ミッションを設計します。

| カテゴリ | スキル数 | 代表的スキル |
|---------|---------|-------------|
| IT基本 | 6 | OS, ネットワーク, データベース, HTML/CSS, Git, 生成AI |
| プログラミング | 2 | メイン開発言語, サブ開発言語 |
| クラウド | 2 | AWS, マルチクラウド |
| コンテナ | 2 | Docker, Kubernetes |
| IaC | 2 | Terraform, その他IaCツール |
| CI/CD | 2 | GitHub Actions, その他CI/CDツール |
| 生成AI | 8 | Copilot, RAG, AIエージェント, ベクトルDB等 |
| DevOps/SRE | 6 | SRE原則, 可観測性, SLI/SLO設計, インシデント管理等 |
| セキュリティ | 5 | セキュアコーディング, DevSecOps, ゼロトラスト等 |
| ML/データ | 4 | 機械学習基礎, MLOps, データパイプライン等 |
| 設計思想 | 4 | DDD, クリーンアーキテクチャ, マイクロサービス等 |
| アーキテクチャ | 3 | API設計, スケーラビリティ設計, DR/BCP設計 |
| ビジネス | 3 | 要件定義, 提案・プレゼン, DX推進 |
| ヒューマンスキル | 16 | タスク管理, コミュニケーション, リーダーシップ等 |

### 2.2 スキル-ミッション紐づけ形式

```yaml
mission:
  title: "SQLインジェクション対策を実装する"
  roadmap:
    skill: "セキュアコーディング"          # RoadmapItem.name
    category: "セキュリティ"               # RoadmapItem.category
    target_level: "L1"                     # 対象レベル
    achievement_criteria: "基本的な脆弱性(SQLi、XSS等)を理解している"  # Lxの定義
```

## 3. 職級別概要

### 3.1 レベル定義

| レベル | 対象者 | 経験年数目安 | ノイズ濃度 | 月間学習時間 |
|-------|-------|-------------|-----------|-------------|
| **L0** | 未経験者 | 0年 | - | 20時間 |
| **L1** | 新人 | 0〜1年 | MINIMAL | 20時間 |
| **L2** | 一人前 | 1〜3年 | MINIMAL→MEDIUM | 20時間 |
| **L3** | 中堅 | 3〜5年 | MEDIUM | 20時間 |
| **L4** | シニア | 5〜10年 | MEDIUM→HIGH | 20時間 |
| **L5** | エキスパート | 10年以上 | HIGH | 20時間 |

### 3.2 月間学習時間配分

| カテゴリ | L0 | L1 | L2 | L3 | L4 | L5 |
|---------|----|----|----|----|----|----|
| **必須研修** | 6h | 6h | 4h | 4h | 4h | 2h |
| **職級別研修** | 10h | 8h | 6h | 4h | 6h | 4h |
| **専門スキル** | 2h | 4h | 6h | 8h | 4h | 6h |
| **自己啓発** | 2h | 2h | 4h | 4h | 6h | 8h |
| **計** | **20h** | **20h** | **20h** | **20h** | **20h** | **20h** |

## 4. ノイズレベル設計

### 4.1 ノイズレベル定義

| ノイズレベル | 教育的意味 | 対象フェーズ |
|-------------|-----------|-------------|
| **MINIMAL** | ノイズ最小限 | L0〜L1 |
| **LOW** | 軽微なノイズ | L1→L2 |
| **MEDIUM** | 中程度のノイズ | L2〜L3 |
| **HIGH** | 高いノイズ | L4〜L5 |
| **MAXIMUM** | 最大ノイズ | L5（特殊ケース） |

### 4.2 ノイズ設計マトリクス

| カテゴリ | 具体的な形態 | 推奨レベル |
|---------|-------------|-----------|
| 情報の不完全性 | 考慮漏れがある仕様書、古いドキュメント | L1〜L3 |
| 技術的負債 | レガシーコード、一貫性のない命名規則 | L2〜L3 |
| 選択肢の過剰 | 複数の解決策、正解のない技術選定 | L3〜L4 |
| コンテキストの揺らぎ | 途中で変わる優先順位、曖昧な要望 | L4〜L5 |
| 検索環境の汚染 | 似た名前の別技術、廃止されたAPI | 全レベル |

## 5. カリキュラム種別と時間

### 5.1 種別定義

| 種別 | 標準時間 | 構成 | 用途 |
|------|---------|------|------|
| **ショート** | 30分 | LESSON×2 | 概念理解、座学 |
| **スタンダード** | 1.5時間 | LESSON×3 + EXERCISE + QUIZ | 基本スキル習得 |
| **ロング** | 3時間 | LESSON×3 + EXERCISE×2 + QUIZ | 実践スキル習得 |
| **ワークショップ** | 4時間 | EXERCISE×3 + 振り返り | ハンズオン中心 |

### 5.2 月間カリキュラム数の目安

| 職級 | ショート | スタンダード | ロング | 合計時間 |
|------|---------|-------------|--------|---------|
| L0 | 10本 | 6本 | 2本 | 20時間 |
| L1 | 8本 | 6本 | 2本 | 20時間 |
| L2 | 6本 | 6本 | 3本 | 20時間 |
| L3 | 4本 | 6本 | 3本 | 20時間 |
| L4 | 4本 | 4本 | 4本 | 20時間 |
| L5 | 2本 | 4本 | 4本 | 20時間 |

## 6. ミッションタイトルパターン

### 6.1 カテゴリ別パターン

| カテゴリ | パターン | 例 |
|---------|---------|-----|
| セキュリティ | 〇〇対策を実装する | SQLインジェクション対策を実装する |
| デバッグ | 〇〇の原因を特定する | バグの原因を特定する |
| 障害対応 | 〇〇を復旧する | システム障害を復旧する |
| 設計 | 〇〇を設計する | 最適なアーキテクチャを設計する |
| 構築 | 〇〇を構築する | CI/CDパイプラインを構築する |
| 移行 | 〇〇に移行する | クラウドに移行する |
| 最適化 | 〇〇を最適化する | パフォーマンスを最適化する |
| 戦略 | 〇〇戦略を策定する | AI活用戦略を策定する |
| 文化 | 〇〇文化を醸成する | DevOps文化を醸成する |
| 学習 | 〇〇を理解する | TCP/IPの基本を理解する |
| 習得 | 〇〇を習得する | 基本コマンドを習得する |

### 6.2 ノイズレベル別タイトル傾向

| ノイズレベル | タイトルの特徴 | 例 |
|-------------|---------------|-----|
| MINIMAL | 明確な目標、正解がある | 「基本コマンドを習得する」 |
| MEDIUM | トレードオフ、選択を含む | 「最適なDB設計を完成する」 |
| HIGH | 曖昧さ、組織調整を含む | 「技術戦略を策定する」 |

## 7. 品質基準

### 7.1 カリキュラム品質チェックリスト

- [ ] **ロードマップ連動**: roadmap.skill, category, target_level, achievement_criteriaが設定されている
- [ ] 対象職級に適切なノイズレベルが設定されている
- [ ] 月間学習時間（20時間）の配分が適切
- [ ] ミッションタイトルが「〜する」形式で具体的
- [ ] 背景ストーリーが没入感を与える
- [ ] ステップが論理的に構成されている
- [ ] 評価基準が明確

### 7.2 年間計画チェックリスト

- [ ] 必須研修がすべてカバーされている
- [ ] **ロードマップの達成基準**が網羅されている
- [ ] 職級昇格に必要なスキルが網羅されている
- [ ] 時間配分が現実的
- [ ] 難易度が段階的に上がる
