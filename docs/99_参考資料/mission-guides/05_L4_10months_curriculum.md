# L4向け 10ヶ月カリキュラム（年間200時間）

## 概要

| 項目 | 内容 |
|-----|------|
| 対象レベル | L4（シニア）→ L5（エキスパート）への成長 |
| 総学習時間 | 200時間（20時間 × 10ヶ月） |
| 形式 | ミッション型カリキュラム |
| ノイズレベル | MEDIUM→HIGH（中〜高） |

---

## 月別詳細

### 月1: 組織全体のCI/CD基盤を設計しよう（20時間）

```yaml
mission:
  title: "組織全体のCI/CD基盤を設計しよう"
  roadmap:
    skills:
      - name: "GitHub Actions"
        category: "CI/CD"
        achievement: "組織全体のCI/CD基盤を設計できる"
      - name: "その他CI/CDツール"
        category: "CI/CD"
        achievement: "CI/CDツールの選定・最適化ができる"
```

| # | ステップタイトル | 種別 | 時間 |
|---|----------------|------|------|
| 1 | 組織のCI/CD現状を分析しよう | LESSON + EXERCISE | 3h |
| 2 | 標準パイプラインを設計しよう | EXERCISE | 4h |
| 3 | セキュリティとコンプライアンスを統合しよう | EXERCISE | 4h |
| 4 | マルチチーム対応のガバナンスを確立しよう | LESSON + EXERCISE | 4h |
| 5 | CI/CD基盤の運用自動化を実装しよう | EXERCISE | 3h |
| 6 | CI/CD基盤設計書を完成させよう | QUIZ + EXERCISE | 2h |

**背景ストーリー**:
組織には20以上のチームがあり、各チームが独自のCI/CDパイプラインを構築している。重複投資、セキュリティの穴、ナレッジの分断...。組織全体で統一された基盤を設計し、開発効率とセキュリティを両立させるミッションだ。
