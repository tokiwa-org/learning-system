# ユビキタス言語定義

本ドキュメントは、人事考課システムで使用する用語を定義し、チーム全体で共通認識を持つためのものです。

---

## 1. 評価に関する用語

| 用語             | 英語               | 定義                                                |
| ---------------- | ------------------ | --------------------------------------------------- |
| **人事考課**     | Performance Review | 年1回（9月）に行う社員の評価プロセス全体            |
| **評価期間**     | Evaluation Period  | 人事考課の対象となる1年間（10月〜翌9月）            |
| **評価サイクル** | Evaluation Cycle   | 1回の人事考課の開始から確定までの一連の流れ         |
| **評価軸**       | Evaluation Axis    | 評価の観点。スキル習得度・職能発揮力・行動貢献の3軸 |
| **評価点**       | Evaluation Score   | 各評価軸の合計点（100点満点）                       |
| **評価ランク**   | Evaluation Rank    | 評価点に基づく5段階評価（S/A/B/C/D）                |
| **自己評価**     | Self Evaluation    | 本人が行う自身の評価入力                            |
| **同僚評価**     | Peer Evaluation    | 同僚が行う行動・貢献に関するフィードバック          |
| **上司評価**     | Manager Evaluation | 直属上司が行う評価入力                              |
| **最終評価**     | Final Evaluation   | 人事部が確定する最終的な評価結果                    |

---

## 2. 給与・昇格に関する用語

| 用語         | 英語                   | 定義                                            |
| ------------ | ---------------------- | ----------------------------------------------- |
| **職級**     | Grade / Level          | 社員のレベル（L1〜L5）。Junior〜Principalに対応 |
| **号俸**     | Step                   | 給与テーブル内の段階（1〜40号俸）               |
| **基礎給**   | Base Salary            | 社歴年数に基づく固定給部分                      |
| **職級給**   | Grade Salary           | 職級×号俸で決まる給与部分                       |
| **昇給**     | Salary Increase        | 号俸が上がること。評価ランクに連動（+0〜+5）    |
| **昇格**     | Promotion              | 職級が上がること（L1→L2など）。10-11月に判定    |
| **昇格試験** | Promotion Exam         | 昇格可否を判定する面談・評価プロセス            |
| **昇格要件** | Promotion Requirements | 昇格に必要な条件（チェックリスト）              |

---

## 3. スキル・ロードマップに関する用語

| 用語                 | 英語               | 定義                                            |
| -------------------- | ------------------ | ----------------------------------------------- |
| **ロードマップ**     | Roadmap            | 各職級で習得すべきスキル・能力の一覧（No.1-82） |
| **ロードマップ項目** | Roadmap Item       | ロードマップ内の個別のスキル・能力項目          |
| **スキルカテゴリ**   | Skill Category     | ロードマップ項目の分類（技術/職能/行動）        |
| **達成状況**         | Achievement Status | 各ロードマップ項目の完了状態                    |
| **達成率**           | Achievement Rate   | 必須項目に対する達成項目の割合                  |
| **必須項目**         | Required Item      | 該当職級で達成が必須とされる項目                |

---

## 4. 承認フローに関する用語

| 用語             | 英語               | 定義                               |
| ---------------- | ------------------ | ---------------------------------- |
| **承認ステップ** | Approval Step      | 評価が確定するまでの各段階         |
| **承認者**       | Approver           | 各ステップで承認を行う担当者       |
| **差戻し**       | Rejection / Return | 前のステップに評価を戻すこと       |
| **承認待ち**     | Pending Approval   | 次の承認者の対応を待っている状態   |
| **確定**         | Finalized          | 全承認が完了し、評価が確定した状態 |

---

## 5. 組織に関する用語

| 用語       | 英語          | 定義                               |
| ---------- | ------------- | ---------------------------------- |
| **社員**   | Employee      | システムの利用者となる従業員       |
| **上司**   | Manager       | 評価対象者の直属の上長             |
| **同僚**   | Peer          | 評価対象者と同じチーム・部署の社員 |
| **人事部** | HR Department | 評価制度の管理・最終承認を行う部門 |
| **部署**   | Department    | 組織上の所属単位                   |

## 6. システム用語

| 用語                 | 英語                | 定義                                       |
| -------------------- | ------------------- | ------------------------------------------ |
| **評価基準**         | Evaluation Criteria | 評価期間ごとに定義される評価軸・比重の設定 |
| **スナップショット** | Snapshot            | 特定時点での基準・データの保存             |

## 7. 学習・育成に関する用語

| 用語             | 英語                 | 定義                                                                                  |
| ---------------- | -------------------- | ------------------------------------------------------------------------------------- |
| **シナリオ**     | Scenario             | 人事部が毎年策定する学習の「お題」。実践的な課題設定                                  |
| **カリキュラム** | Curriculum           | シナリオからLLMで生成される学習ドキュメント。ロードマップ項目の習得に繋がる内容を含む |
| **学習課題**     | Learning Task        | 社員に割り当てられた実施すべきカリキュラム                                            |
| **学習フェーズ** | Learning Phase       | 学習の段階。フィルタリング学習→リファクタリング学習→アーキテクチャ学習の3段階         |
| **ノイズ**       | Noise                | 教育的ノイズ。実務で遭遇する不正確・誤解を招く情報を意図的に含めたもの                |
| **ノイズ識別**   | Noise Identification | 学習者がノイズを正しく見分けられたかの評価                                            |
| **習得判定**     | Mastery Judgment     | カリキュラム完了時に対応するロードマップ項目が習得済みになる判定                      |

## 8. 評価ランクと号俸アップの対応

| 評価点   | 評価ランク | 意味             | 号俸アップ |
| -------- | ---------- | ---------------- | ---------- |
| 90点以上 | S          | 卓越した成果     | +5         |
| 75点以上 | A          | 期待を上回る     | +3         |
| 60点以上 | B          | 期待通り（標準） | +2         |
| 45点以上 | C          | 期待を下回る     | +1         |
| 45点未満 | D          | 大幅な改善が必要 | ±0         |

## 9. 職級の定義

| 職級 | 名称        | 役割・期待                                   |
| ---- | ----------- | -------------------------------------------- |
| L1   | Entry-level | 指示のもと作業可能。基礎スキルの習得段階     |
| L2   | Mid-level   | 自走で業務遂行可能。一通りの実務を任せられる |
| L3   | Senior      | 複雑な課題解決が可能。設計・後輩指導ができる |
| L4   | Lead        | チームを牽引。組織的な課題解決をリード       |
| L5   | Manager     | 組織全体を管理。戦略決定と人材育成を担う     |

## 10. 承認フロー

```
本人（自己評価）
    ↓ 提出
同僚（行動評価+コメント）
    ↓ フィードバック完了
上司（全軸評価+承認）
    ↓ 承認 or 差戻し
人事部（最終確認+確定）
    ↓ 確定
【評価確定】
```
